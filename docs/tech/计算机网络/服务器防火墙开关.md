### 🛡️ **主流防火墙系统**

#### **1. iptables (传统Linux防火墙)**

##### **查看状态**
```bash
# 查看所有规则
sudo iptables -L -n -v

# 查看具体链的规则
sudo iptables -L INPUT -n -v
sudo iptables -L OUTPUT -n -v

# 查看规则编号（便于删除）
sudo iptables -L --line-numbers
```

##### **临时开关**
```bash
# 完全关闭防火墙（清空所有规则，允许所有流量）
sudo iptables -F          # 清空所有规则
sudo iptables -X          # 删除用户自定义链
sudo iptables -P INPUT ACCEPT    # 设置默认策略为接受
sudo iptables -P FORWARD ACCEPT
sudo iptables -P OUTPUT ACCEPT

# 完全开启防火墙（默认拒绝所有）
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# 保存规则（避免重启后丢失）
sudo iptables-save > /etc/iptables/rules.v4
```

##### **常用规则管理**
```bash
# 开放端口
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT    # 开放SSH
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT    # 开放HTTP
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT   # 开放HTTPS

# 禁止特定IP
sudo iptables -A INPUT -s 192.168.1.100 -j DROP

# 删除规则（需要先查看行号）
sudo iptables -D INPUT 3   # 删除INPUT链的第3条规则
```

#### **2. firewalld (CentOS/RHEL 7+ & Fedora)**

##### **服务管理**
```bash
# 查看状态
sudo systemctl status firewalld
sudo firewall-cmd --state

# 启动防火墙
sudo systemctl start firewalld

# 停止防火墙
sudo systemctl stop firewalld

# 重启防火墙
sudo systemctl restart firewalld

# 开机禁用
sudo systemctl disable firewalld

# 开机启用
sudo systemctl enable firewalld
```

##### **运行时 vs 永久配置**
```bash
# 运行时配置（立即生效，重启后丢失）
sudo firewall-cmd --add-port=80/tcp

# 永久配置（重启后依然有效）
sudo firewall-cmd --add-port=80/tcp --permanent
sudo firewall-cmd --reload   # 重新加载使永久配置生效
```

##### **常用操作**
```bash
# 查看所有信息
sudo firewall-cmd --list-all

# 开放端口
sudo firewall-cmd --add-port=80/tcp --permanent
sudo firewall-cmd --add-port=443/tcp --permanent
sudo firewall-cmd --add-port=22/tcp --permanent

# 移除端口
sudo firewall-cmd --remove-port=8080/tcp --permanent

# 开放服务（预定义的服务）
sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --add-service=https --permanent
sudo firewall-cmd --add-service=ssh --permanent

# 重新加载配置
sudo firewall-cmd --reload
```

#### **3. ufw (Ubuntu/Debian)**

##### **服务管理**
```bash
# 查看状态
sudo ufw status
sudo ufw status verbose   # 详细状态
sudo ufw status numbered  # 带编号的状态

# 启用防火墙
sudo ufw enable

# 禁用防火墙
sudo ufw disable

# 重置防火墙
sudo ufw reset
```

##### **规则管理**
```bash
# 开放端口
sudo ufw allow 22/tcp     # SSH
sudo ufw allow 80/tcp     # HTTP
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 8080       # 同时开放TCP和UDP

# 按服务名开放
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https

# 限制特定IP
sudo ufw allow from 192.168.1.100
sudo ufw allow from 192.168.1.0/24

# 删除规则
sudo ufw delete allow 80/tcp
sudo ufw delete 3         # 删除编号为3的规则
```

### 🔧 **快速开关脚本**

#### **一键防火墙管理脚本**
```bash
#!/bin/bash
# firewall_manager.sh

case "$1" in
    "status")
        echo "=== 防火墙状态 ==="
        # 检测使用的防火墙类型
        if command -v ufw &> /dev/null && sudo ufw status | grep -q "active"; then
            echo "防火墙: ufw"
            sudo ufw status verbose
        elif systemctl is-active --quiet firewalld; then
            echo "防火墙: firewalld"
            sudo firewall-cmd --state
            sudo firewall-cmd --list-all
        elif sudo iptables -L -n | grep -q "Chain INPUT"; then
            echo "防火墙: iptables"
            sudo iptables -L -n -v
        else
            echo "未检测到活动的防火墙"
        fi
        ;;
        
    "start")
        echo "启动防火墙..."
        if command -v ufw &> /dev/null; then
            sudo ufw enable
        elif command -v firewall-cmd &> /dev/null; then
            sudo systemctl start firewalld
        else
            echo "使用iptables，需要手动配置规则"
        fi
        ;;
        
    "stop")
        echo "停止防火墙..."
        if command -v ufw &> /dev/null; then
            sudo ufw disable
        elif command -v firewall-cmd &> /dev/null; then
            sudo systemctl stop firewalld
        else
            sudo iptables -F
            sudo iptables -X
            sudo iptables -P INPUT ACCEPT
            sudo iptables -P FORWARD ACCEPT
            sudo iptables -P OUTPUT ACCEPT
        fi
        ;;
        
    "restart")
        echo "重启防火墙..."
        if command -v ufw &> /dev/null; then
            sudo ufw disable
            sudo ufw enable
        elif command -v firewall-cmd &> /dev/null; then
            sudo systemctl restart firewalld
        else
            echo "请手动重置iptables规则"
        fi
        ;;
        
    *)
        echo "用法: $0 {status|start|stop|restart}"
        echo "  status  查看防火墙状态"
        echo "  start   启动防火墙"
        echo "  stop    停止防火墙" 
        echo "  restart 重启防火墙"
        ;;
esac
```

**使用方法**：
```bash
chmod +x firewall_manager.sh
./firewall_manager.sh status
./firewall_manager.sh stop
./firewall_manager.sh start
```

### 🎯 **操作系统特定命令**

```bash
# 检查防火墙类型
systemctl status firewalld
systemctl status iptables

# 切换防火墙系统
systemctl stop iptables
systemctl disable iptables

sudo ufw default deny incoming   # 默认拒绝所有入站
sudo ufw default allow outgoing  # 默认允许所有出站
sudo ufw allow ssh
sudo ufw enable
```

```bash
# Amazon Linux 2通常使用firewalld
sudo systemctl status firewalld
sudo firewall-cmd --list-all
```

### 🔍 **防火墙诊断命令**

#### **检查端口是否真正开放**
```bash
# 从外部测试（在另一台服务器上）
telnet 你的服务器IP 22
nc -zv 你的服务器IP 80

# 在本地测试
sudo netstat -tulpn | grep :80
sudo ss -tulpn | grep :80
```

#### **查看防火墙日志**
```bash
# ufw日志
sudo tail -f /var/log/ufw.log

# firewalld日志
sudo journalctl -u firewalld -f

# iptables日志（如果配置了日志规则）
sudo tail -f /var/log/kern.log | grep iptables
```

### ⚠️ **重要安全提醒**

#### **远程操作防火墙的危险**
```bash
# ❌ 危险操作 - 可能导致SSH连接断开
sudo ufw disable
sudo systemctl stop firewalld

# ✅ 安全操作 - 先放行SSH端口
sudo ufw allow ssh
sudo ufw enable

# 或者使用nohup在后台执行
nohup sudo ufw disable &
```

#### **紧急恢复方法**
如果误操作导致无法SSH连接：
1. 通过云平台的控制台登录
2. 使用VNC或串行控制台
3. 重启服务器进入单用户模式
