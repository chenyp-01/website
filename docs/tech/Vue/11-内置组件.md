## 1. 动态组件
### 语法
有些场景会需要在两个组件间来回切换，比如 Tab 界面, 可通过<component>动态组件实现
```vue
<!-- Home为组件名 -->
<component is="Home"></component>
<component :is="'Home'"></component>
<!-- currentTab 改变时组件也改变 -->
<component :is="currentTab"></component>
```
### 案例
Home.vue
```vue
<template>
    <div class="tab">
        Home component
    </div>
</template>
```
Posts.vue
```vue
<template>
    <div class="tab">
        Posts component
    </div>
</template>
```
Archive.vue
```vue
<template>
    <div class="tab">
        Archive component
    </div>
</template>
```
App.vue
```vue
<script>
import Home from './Home.vue'
import Posts from './Posts.vue'
import Archive from './Archive.vue'
  
export default {
  components: {
    Home,
    Posts,
    Archive
  },
  data() {
    return {
      currentTab: 'Home',
      tabs: ['Home', 'Posts', 'Archive']
    }
  }
}
</script>

<template>
  <div class="demo">
    <button
       v-for="tab in tabs"
       :key="tab"
       :class="['tab-button', { active: currentTab === tab }]"
       @click="currentTab = tab"
     >
      {{ tab }}
    </button>
	  <component :is="currentTab" class="tab"></component>
  </div>
</template>

<style>

  border: 1px solid #ccc;

  margin-bottom: -1px;
  margin-right: -1px;

  background: #e0e0e0;

  background: #e0e0e0;

```
## 2. Transition组件
Vue 提供了两个内置组件，可以帮助你制作基于状态变化的过渡和动画：
### Transition 组件
#### 认识Transition组件
Transition 是一个内置组件，这意味着它在任意别的组件中都可以被使用，无需注册。它可以将进入和离开动画应用到通过默认插槽传递给它的元素或组件上。进入或离开可以由以下的条件之一触发：

- 由 v-if 所触发的切换
- 由 v-show 所触发的切换
- 由特殊元素 <component> 切换的动态组件
- 改变特殊的 key 属性
#### 基础用法
```vue
<button @click="show = !show">Toggle</button>
<Transition name="fade">
  <p v-if="show">hello</p>
</Transition>
```
```css
.fade-enter-active,
.fade-leave-active {
  transition: opacity 0.5s ease;
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}
```
**TIP**
**<Transition> 仅支持单个元素或组件作为其插槽内容。如果内容是一个组件，这个组件必须仅有一个根元素。**
当一个 <Transition> 组件中的元素被插入或移除时，会发生下面这些事情：
Vue 会自动检测目标元素是否应用了 CSS 过渡或动画。如果是，则一些 [CSS 过渡 class](https://cn.vuejs.org/guide/built-ins/transition.html#transition-classes) 会在适当的时机被添加和移除。

#### CSS过渡class

#### CSS的animation
```vue

  <p v-if="show" style="text-align: center;">
    Hello here is some bouncy text!
  </p>
</Transition>
```
```css
.bounce-enter-active {
  animation: bounce-in 0.5s;
}

}

```
#### 可复用过渡效果
得益于 Vue 的组件系统，过渡效果是可以被封装复用的。要创建一个可被复用的过渡，我们需要为 <Transition> 组件创建一个包装组件，并向内传入插槽内容
```vue
<!-- MyTransition.vue -->
<script>
</script>

<template>
  <!-- 包装内置的 Transition 组件 -->
  <Transition
    name="my-transition"
    <slot></slot> <!-- 向内传递插槽内容 -->
  </Transition>
</template>

<style>
/*
  必要的 CSS...
  注意：避免在这里使用 <style scoped>
  因为那不会应用到插槽内容上
*/
</style>
```
使用自定义过渡组件
```vue
<MyTransition>
  <div v-if="show">Hello</div>
</MyTransition>
```
#### 出现时过渡
如果你想在某个节点初次渲染时应用一个过渡效果，你可以添加 appear prop：
```vue
<Transition appear>
  
</Transition>
```
#### 过渡模式
先执行离开动画，然后在其完成**之后**再执行元素的进入动画
```vue
<Transition mode="out-in">
  
</Transition>
```
#### 组件间过渡
<Transition> 也可以作用于[动态组件](https://cn.vuejs.org/guide/essentials/component-basics.html#dynamic-components)之间的切换
```vue
<Transition name="fade" mode="out-in">
  <component :is="activeComponent"></component>
</Transition>
```
#### 动态过渡
```vue
<Transition :name="transitionName">
  
</Transition>
```
### TransitionGroup 组件
会在一个 v-for 列表中的元素或组件被插入，移动，或移除时应用动画。
```vue

<script>
import { shuffle } from 'lodash'

const getInitialItems = () => [1, 2, 3, 4, 5]
let id = getInitialItems().length + 1

export default {
  data() {
    return {
      items: getInitialItems()
    }
  },
  methods: {
    insert() {
      const i = Math.round(Math.random() * this.items.length)
      this.items.splice(i, 0, id++)
    },
    reset() {
      this.items = getInitialItems()
    },
    shuffle() {
      this.items = shuffle(this.items)
    },
    remove(item) {
      const i = this.items.indexOf(item)
      if (i > -1) {
        this.items.splice(i, 1)
      }
    }
  }
}
</script>

<template>
  <button @click="insert">insert at random index</button>
  <button @click="reset">reset</button>
  <button @click="shuffle">shuffle</button>

  <TransitionGroup tag="ul" name="fade" class="container">
    <div v-for="item in items" class="item" :key="item">
      {{ item }}
      <button @click="remove(item)">x</button>
    </div>
  </TransitionGroup>
</template>

<style>
.container {
  position: relative;
  padding: 0;
}

.item {
  width: 100%;
  height: 30px;
  background-color: #f3f3f3;
  border: 1px solid #666;
  box-sizing: border-box;
}

/* 1. 声明过渡效果 */
.fade-move,
.fade-enter-active,
.fade-leave-active {
  transition: all 0.5s cubic-bezier(0.55, 0, 0.1, 1);
}

/* 2. 声明进入和离开的状态 */
.fade-enter-from,
.fade-leave-to {
  opacity: 0;
  transform: scaleY(0.01) translate(30px, 0);
}

/* 3. 确保离开的项目被移除出了布局流
      以便正确地计算移动时的动画效果。 */
.fade-leave-active {
  position: absolute;
}
</style>
```
### Vue2和Vue3的区别

Vue2

```css
/* vue2的类名没有from和to */
.fade-enter,
.fade-leave{
  opacity: 0;
}
```

Vue3

```css
/* vue3需要添加from和to*/
.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}
```



## 3. KeepAlive组件 （重要）

<KeepAlive> 是一个内置组件，它的功能是在多个组件间动态切换时缓存被移除的组件实例。

### 案例
```vue
<!-- 非活跃的组件将会被缓存！ -->
<KeepAlive>
  <component :is="activeComponent" />
</KeepAlive>
```
### 包含/排除

> include和exclude中的标识符为组件定义中的name选项的值

```vue
<!-- 以英文逗号分隔的字符串 -->

<!-- 包含 -->
<KeepAlive include="a,b">
  <component :is="view" />
</KeepAlive>

<!-- 排除 -->
<KeepAlive exclude="a,b">
  <component :is="view" />
</KeepAlive>

<!-- 正则表达式 (需使用 `v-bind`) -->
<KeepAlive :include="/a|b/">
  <component :is="view" />
</KeepAlive>

<!-- 数组 (需使用 `v-bind`) -->
<KeepAlive :include="['a', 'b']">
  <component :is="view" />
</KeepAlive>
```
### 最大缓存实例数
如果缓存的实例数量即将超过指定的那个最大数量，则最久没有被访问的缓存实例将被销毁，以便为新的实例腾出空间。
```vue
<KeepAlive :max="10">
  <component :is="activeComponent" />
</KeepAlive>
```
### 缓存实例的生命周期
一个持续存在的组件可以通过 [activated](https://cn.vuejs.org/api/options-lifecycle.html#activated) 和 [deactivated](https://cn.vuejs.org/api/options-lifecycle.html#deactivated) 选项来注册相应的两个状态的生命周期钩子
```javascript
export default {
  activated() {
    // 在首次挂载、
    // 以及每次从缓存中被重新插入的时候调用
  },
  deactivated() {
    // 在从 DOM 上移除、进入缓存
    // 以及组件卸载时调用
  }
}
```
这两个钩子不仅适用于 <KeepAlive> 缓存的根组件，也适用于缓存树中的后代组件
## 4. Teleport组件
** vue2没有Teleport组件



`<Teleport>` 是一个内置组件，它可以将一个组件内部的一部分模板“传送”到该组件的 DOM 结构外层的位置去。

### 应用场景

有时我们可能会遇到这样的场景：一个组件模板的一部分在逻辑上从属于该组件，但从整个应用视图的角度来看，它在 DOM 中应该被渲染在整个 Vue 应用外部的其他地方。

这类场景最常见的例子就是全屏的模态框。理想情况下，我们希望触发模态框的按钮和模态框本身是在同一个组件中，因为它们都与组件的开关状态有关。但这意味着该模态框将与按钮一起渲染在应用 DOM 结构里很深的地方。这会导致该模态框的 CSS 布局代码很难写。

### 全屏模态框案例
Parent.vue
```vue
<script>
import Modal from './Modal.vue'

export default {
  components: {
    Modal
  },
  data() {
    return {
      showModal: false
    }
  }
}
</script>

<template>
  <div class="top">
    <button id="show-modal" @click="showModal = true">Show Modal</button>
    <Teleport to="body">
    <!-- 使用这个 modal 组件，传入 prop -->
    <modal :show="showModal" @close="showModal = false">
      <template #header>
        <h3>custom header</h3>
      </template>
    </modal>
    </Teleport>
  </div>
</template>

<style lang="scss" scoped>
.top {
  /*--------------------------------注意transform属性会影响子组件position:fixed的定位*/
  transform: translateX(300px);
  width: 400px;
  height: 300px;
  border: 1px solid #000;
}
</style>
```
Modal.vue
```vue
<script>
export default {
  props: {
    show: Boolean
  }
}
</script>

<template>
  <Transition name="modal">
    <div v-if="show" class="modal-mask">

        <div class="modal-header">
          <slot name="header">default header</slot>

          <slot name="body">default body</slot>

        <div class="modal-footer">
          <slot name="footer">
            default footer
            <button
              class="modal-default-button"
              @click="$emit('close')"
            >OK</button>
          </slot>
        </div>
      </div>
    </div>
  </Transition>
</template>

<style>

  position: fixed;
  z-index: 9998;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background-color: rgba(0, 0, 0, 0.5);
  display: flex;

}

  width: 300px;
  margin: auto;
  padding: 20px 30px;
  background-color: #fff;
  border-radius: 2px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.33);
  transition: all 0.3s ease;
}

.modal-header h3 {
  margin-top: 0;

}

/*

 * 你可以简单地通过编辑这些样式
```

### **🚀 Vue3 `Teleport` 详解**

`<Teleport>` 是 Vue3 提供的 **内置组件**，用于**将组件内容渲染到 DOM 结构的其他位置**，而不影响 Vue 的响应式系统。

------

## **1. 为什么需要 `Teleport`？**

在 Vue 组件中，默认情况下所有内容都会被渲染在当前组件的 DOM 结构内。但在某些场景下，我们需要 **让组件的内容脱离当前 DOM 结构，挂载到 `body` 或其他指定元素**，比如：

- **模态框（Modal）**
- **通知（Toast）**
- **悬浮层（Dropdown、Popover）**
- **全局 Loading 遮罩层**

Vue3 之前，我们一般用 **`appendChild()` 手动插入 DOM**，但 `Teleport` 让这个过程变得更优雅。

------

## **2. 语法**

```vue
<Teleport to="目标容器选择器">
  <div>要传送的内容</div>
</Teleport>
```

- `to="目标选择器"`：指定要把内容**传送到的 DOM 位置**
- `<Teleport>` 内部的内容会**从组件内部移出，渲染到指定的 `to` 目标内**，但不会影响 Vue 的响应式功能

------

## **3. 示例**

### ✅ **示例 1：模态框（Modal）**

```vue
<template>
  <button @click="showModal = true">打开模态框</button>

  <Teleport to="body">
    <div v-if="showModal" class="modal">
      <div class="modal-content">
        <h2>模态框</h2>
        <p>这是一个使用 Teleport 的模态框</p>
        <button @click="showModal = false">关闭</button>
      </div>
    </div>
  </Teleport>
</template>

<script setup>
import { ref } from 'vue'

const showModal = ref(false)
</script>

<style>
.modal {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: rgba(0, 0, 0, 0.5);
  display: flex;
  justify-content: center;
  align-items: center;
}

.modal-content {
  background: white;
  padding: 20px;
  border-radius: 8px;
}
</style>
```

### **📌 解析**

- `<Teleport to="body">`：模态框的内容会直接被渲染到 `body` 下，而不会受组件嵌套影响
- **避免 z-index 层级问题**，例如如果组件嵌套太深，模态框可能被 `overflow: hidden` 限制，而 `Teleport` 直接放到 `body` 下可以避免这些问题

------

### ✅ **示例 2：通知（Toast）**

```vue
<template>
  <button @click="showToast = true">显示通知</button>

  <Teleport to="#app">
    <div v-if="showToast" class="toast">通知消息</div>
  </Teleport>
</template>

<script setup>
import { ref } from 'vue'

const showToast = ref(false)

setTimeout(() => {
  showToast.value = false
}, 3000)
</script>

<style>
.toast {
  position: fixed;
  bottom: 20px;
  right: 20px;
  background: #333;
  color: white;
  padding: 10px;
  border-radius: 5px;
}
</style>
```

### **📌 解析**

- **避免受父组件 CSS 影响**，比如 `overflow: hidden`
- **更方便全局管理通知消息**，如 Vuex / Pinia 统一管理

------

## **4. `Teleport` 的作用范围**

- **`Teleport` 只会移动 DOM 结构，不会影响 Vue 组件的响应式状态**
- **父子组件关系不变**，即使 `Teleport` 传送到 `body`，它的 `props`、`data` 依然来自原组件
- **可以传送到任何合法的 DOM 选择器（如 `body`, `#app`, `.container`）**

------

## **5. 什么时候不用 `Teleport`？**

🚫 **普通组件嵌套不需要 `Teleport`**

```vue
<template>
  <div class="parent">
    <ChildComponent />
  </div>
</template>
```

👉 这种情况下，子组件不需要脱离 `parent`，所以不需要 `Teleport`。

🚫 **组件本身不需要脱离当前 DOM 结构** 比如 **普通卡片、列表、按钮等 UI 组件**，没有 `overflow: hidden` 或 `z-index` 问题时，不需要 `Teleport`。

------

## **6. `Teleport` vs. `appendChild()`**

| **方式**          | **Vue3 Teleport** 🏆            | **手动 `appendChild()`** ❌            |
| ----------------- | ------------------------------ | ------------------------------------- |
| **维护方便**      | ✅ 组件内管理，不依赖 DOM 操作  | ❌ 需要手写 `document.createElement()` |
| **自动回收**      | ✅ 组件销毁时自动删除           | ❌ 需要手动 `removeChild()`            |
| **响应式支持**    | ✅ Vue3 响应式                  | ❌ 需手动管理状态                      |
| **避免 CSS 问题** | ✅ 不受 `overflow: hidden` 限制 | ❌ 可能被 `z-index` 影响               |

------

## **7. 总结**

✅ **`Teleport` 作用**

- **脱离组件层级**，将内容渲染到**其他 DOM 位置**
- 适用于 **模态框（Modal）**、**通知（Toast）**、**下拉菜单（Dropdown）** 等
- **不影响 Vue 的响应式**，依然保留组件的响应式状态

✅ **最佳使用场景**

- **模态框（Modal）**
- **通知（Toast）**
- **下拉菜单（Dropdown）**
- **全局 Loading 遮罩层**
- **Tooltip（悬浮提示）**

🚀 `Teleport` 让 Vue3 组件更灵活、更易维护，避免手写 `appendChild()` 操作。

**你有什么实际业务场景需要用到 `Teleport` 吗？可以一起探讨下最佳实现方案！😊**

## **Vue3 `Suspense` 详解 🚀**

`<Suspense>` 是 Vue3 提供的一个**内置组件**，主要用于**异步组件加载时的占位内容**，可以**在等待异步数据加载时显示 fallback（备用）内容**，比如：

- **懒加载组件时显示加载动画**
- **数据获取完成前显示骨架屏**
- **等异步请求完成后再渲染页面**

------

## **1. `Suspense` 语法**

```vue
<Suspense>
  <template #default>
    <AsyncComponent />
  </template>
  <template #fallback>
    <p>加载中...</p>
  </template>
</Suspense>
```

### **📌 说明**

- `#default` 里面放**需要等待加载的组件**
- `#fallback` 里面放**加载时的占位内容**（如 "加载中..." 或骨架屏）

------

## **2. 为什么需要 `Suspense`？**

在 Vue2 或 Vue3 中，**异步组件**可以用 `defineAsyncComponent()` 定义：

```js
import { defineAsyncComponent } from 'vue';

const AsyncComponent = defineAsyncComponent(() =>
  import('./MyComponent.vue')
);
```

但如果这个组件依赖**异步数据**，比如 `fetch()` 获取接口数据，Vue2 需要**手动处理加载状态**：

```vue
<template>
  <div>
    <p v-if="loading">加载中...</p>
    <p v-else>{{ data }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return { data: null, loading: true };
  },
  async mounted() {
    this.data = await fetchData();
    this.loading = false;
  }
};
</script>
```

👉 **问题**：

- 需要**手动管理 `loading` 状态**
- 代码冗余，每个组件都要加 `loading` 逻辑

**Vue3 的 `<Suspense>`** 让这些变得更简单，**无需额外 `loading` 变量，Vue 会自动等待组件的 `setup()` 逻辑执行完毕**！

------

## **3. `Suspense` 结合异步组件**

### **✅ 示例：异步加载组件**

#### **📌 组件 `AsyncComponent.vue`（有异步逻辑）**

```vue
<script setup>
import { ref } from 'vue';

const data = ref(null);

setTimeout(() => {
  data.value = '异步数据加载完成！';
}, 2000);
</script>

<template>
  <div>
    <p v-if="!data">数据加载中...</p>
    <p v-else>{{ data }}</p>
  </div>
</template>
```

------

### **✅ `Suspense` 组件使用**

```vue
<template>
  <Suspense>
    <template #default>
      <AsyncComponent />
    </template>
    <template #fallback>
      <p>组件加载中...</p>
    </template>
  </Suspense>
</template>

<script setup>
import AsyncComponent from './AsyncComponent.vue';
</script>
```

### **📌 运行逻辑**

1. **Vue 先渲染 `<Suspense>` 的 `#fallback` 部分** (`组件加载中...`)
2. **等待 `AsyncComponent.vue` 的 `setup()` 执行完毕**
3. **数据加载完成后，替换 `#default` 里的 `AsyncComponent.vue`**

👉 **不需要手动 `loading` 状态，Vue 自动管理异步渲染！**

------

## **4. `Suspense` + `async setup()`**

如果组件的 `setup()` 函数是 `async` 的，Vue 会**自动等待它执行完**。

### **✅ 示例：使用 `async setup()` 处理异步数据**

#### **📌 `AsyncComponent.vue`**

```vue
<script setup>
import { ref } from 'vue';

const data = ref(null);

await new Promise(resolve => setTimeout(resolve, 2000)); // 模拟接口请求
data.value = '异步数据加载完成！';
</script>

<template>
  <div>{{ data }}</div>
</template>
```

#### **📌 `App.vue`**

```vue
<Suspense>
  <template #default>
    <AsyncComponent />
  </template>
  <template #fallback>
    <p>正在加载数据...</p>
  </template>
</Suspense>
```

👉 Vue 会**自动等待 `async setup()` 执行完毕**，然后替换 `fallback` 内容。

------

## **5. `Suspense` 的作用范围**

- `Suspense` **不会影响普通组件**，只作用于**异步组件**
- **如果 `#default` 内有多个组件**，**只要有一个是异步的，整个 Suspense 都会等待**
- **`Suspense` 只会生效一次**，组件加载后不会再次触发 `fallback`

------

## **6. `Suspense` vs `defineAsyncComponent()`**

| **特性**                | **`Suspense`** 🏆               | **`defineAsyncComponent()`** ❌ |
| ----------------------- | ------------------------------ | ------------------------------ |
| **作用范围**            | 作用于整个异步组件             | 只作用于 `import()` 组件       |
| **数据请求**            | `setup()` 里的异步数据也会等待 | 只等待组件 `import()`          |
| **是否可用 `fallback`** | ✅ 组件加载 & 数据获取都可用    | 只能用于懒加载组件             |
| **使用方式**            | `<Suspense>` 组件包裹          | 需要 `defineAsyncComponent()`  |

📌 **简单理解**：

- `defineAsyncComponent()` 主要用于 **懒加载组件**
- `Suspense` 既能等待组件 `import()`，还能等待 `setup()` 里的**异步数据**

------

## **7. `Suspense` 适用场景**

✅ **适合使用 `Suspense` 的情况**

- **需要在组件异步加载时显示加载状态**
- **组件 `setup()` 需要异步获取数据**
- **整个页面需要等待多个异步组件完成**

🚫 **不适合使用 `Suspense` 的情况**

- **普通组件（不涉及异步加载）**
- **希望控制多个组件的加载顺序**（需要手动管理状态）
- **希望手动控制 `loading` 逻辑**

------

## **8. `Suspense` 配合 `async setup()` + API 请求**

### **✅ 示例：结合 `fetch()` 请求 API**

```vue
<script setup>
import { ref } from 'vue';

const data = ref(null);

async function fetchData() {
  const res = await fetch('https://jsonplaceholder.typicode.com/todos/1');
  data.value = await res.json();
}

await fetchData();
</script>

<template>
  <div v-if="data">
    <h3>{{ data.title }}</h3>
  </div>
</template>
```

📌 **这样写，Vue 会等 `fetchData()` 完成后才渲染组件！**

## **9. 总结**

### ✅ **`Suspense` 作用**

1. **在异步组件加载时显示 `fallback` 内容**
2. **等待 `setup()` 里的异步操作执行完毕**
3. **避免手写 `loading` 变量，Vue 自动管理状态**
4. **适用于 `async setup()` + API 请求**

### ✅ **最佳实践**

- **`Suspense` 适用于组件级别的异步等待**
- **推荐在 `setup()` 里使用 `async/await`**
- **适用于懒加载组件、模态框、API 数据获取等场景**

------

## **10. `Suspense` 相关问题**

❓ **如果组件加载失败会怎么样？**

- `Suspense` 只会等待组件，如果组件 `setup()` 抛出错误，会直接报错（可以用 `try-catch` 处理）

❓ **如何让多个 `Suspense` 组件同时加载？**

- 需要**独立管理多个异步组件的 `loading` 状态**，可以用 `Promise.all()` 并行加载数据
