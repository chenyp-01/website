# 2. 编程式导航和传参
## 声明式导航
使用 <router-link> 创建 a 标签来定义导航链接
```vue
<router-link to="/list">列表页</router-link>
<router-link to="/list?id=2">列表页</router-link>
<!--下面这种传参方式，必须和动态路由配置相配合-->
<router-link :to="`/list/${id}`">列表页</router-link>

<!--下面这种传参方式 已废弃-->
<router-link :to="{name:'路由名称',params:{}}">列表页</router-link>

<router-link :to="{name:'路由名称',query:{}}">列表页</router-link>
<router-link :to="{path:'/list',query:{}}">列表页</router-link>
```
## 编程式导航
借助 router 的实例方法，通过编写代码来实现。
### 获取路由对象
```vue
<script setup>
  //引入useRouter钩子函数
  import {useRouter} from 'vue-router'
  //获取路由对象
  const router = useRouter()
</script>
```
### 实现导航的方法

- router.push() 

向 history 栈添加一个新的记录，所以，当用户点击浏览器后退按钮时，会回到之前的 URL
当点击 <router-link> ，内部会调用这个方法， <router-link :to="..."> 相当于调用 router.push()

- router.replace()

它的作用类似于 router.push，唯一不同的是，它在导航时不会向 history 添加新记录，正如它的名字所暗示的那样——它取代了当前的条目

- router.go()
```javascript
// 字符串路径,等价于 <router-link to='/users/参数'>
router.push('/users/eduardo')

// 带有路径的对象 等价于 <router-link to='/users/参数'>
router.push({ path: '/users/eduardo' })

// 带查询参数，结果是  等价于 <router-link to='/register?plan=private'>
router.push({ path: '/register', query: { plan: 'private' } })

// 相当于 <router-link :to="..." replace>
router.push({ path: '/home', replace: true })

router.replace({ path: '/home' })
```
> 注意： 
>
> 1. name为路由配置中命名的路由名称

## 接收路由传参
```javascript
import {useRoute} from 'vue-router'
const route = useRoute()
console.log(route.params.XX)  //动态路由传参
console.log(route.query.XX) // url?后面的传参或query传参
```
## Vue2
```vue
<script>
  ...
  //编程式导航
  this.$router.push()
  // 接收导航参数
  this.$route.params.XX
  this.$route.query.XX
  ...
</script>

<template>
   <button @click="$router.push()">导航</button>
   {{$route.params.XX}}
</template>
```
# 3.历史记录模式
在创建路由器实例时，history 配置允许我们在不同的历史模式中进行选择。
## Hash模式
hash 模式是用 createWebHashHistory() 创建的：
```javascript
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    //...
  ],
})
```
它在内部传递的实际 URL 之前使用了一个哈希字符（#）。由于这部分 URL 从未被发送到服务器，所以它不需要在服务器层面上进行任何特殊处理。
## HTML5模式
用 createWebHistory() 创建 HTML5 模式，推荐使用这个模式：
```javascript
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    //...
  ],
})
```
## 面试题：Hash模式和HTML5模式的区别
### Hash模式

- 地址栏中有# 
- 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载
### HTML5模式

- 地址栏没有#, 更好看一些
- 如果不想要很丑的 hash，我们可以用路由的 HTML5 模式，这种模式充分利history.pushState API 来完成 URL 跳转而无须重新加载页面。
- 但这种模式以后上线，需要后端在服务器上配置才能使用 
## Vue2
```javascript
// 改变模式,添加mode选项，默认是hash
const router = new VueRouter({
    routes,
    // mode: 'history'
    mode: 'hash'
})
```

# 5.路由元信息
定义路由的时候你可以配置 meta 字段, 将任意信息附加到路由上
## 定义路由元信息 
```javascript
const routes = [
  {
    path: '/circle',
    component: circle
  },
  {
    path: '/cart',
    component: cart,
    meta: {
      // Authorization 授权
      auth: true
    }
  },
  {
    name: 'my',
    path: '/my',
    component: my,
    // 路由元信息，可以在组件中通过this.$route.meta访问到这些信息
    // setup中使用 route
    meta: {
      title: '个人信息',
      icon: 'icon-center',
      auth: true
    }
  },
  {
    path: '/login',
    component: login
  }
]
```
## 组件中接收路由元信息
```javascript
<script setup>
import { onMounted } from 'vue';
import { useRoute } from 'vue-router';
const route = useRoute()
onMounted(() => {
    // route.meta.XX
    console.log('route-meta',route.meta.auth);
})
</script>
```
#  6.导航守卫
又称为路由守卫或路由拦截
> 重点学习全局前置守卫，其它了解即可

`vue-router` 提供的导航守卫主要用来通过跳转或取消的方式守卫导航。有多种机会植入路由导航过程中：全局的, 单个路由独享的, 或者组件级的。

- 全局路由守卫在路由配置文件中定义
- 单个路由独享的守卫，也是在路由配置文件中定义，但是，是在某一个具体的路由定义中添加
- 组件级的路由守卫： 添加在某个组件内部
## 全局前置守卫 
Vue3
每个守卫方法接收两个参数：
to: 即将要进入的目标
from: 当前导航正要离开的路由
可以返回的值如下:
false: 取消当前的导航
一个[路由地址](https://router.vuejs.org/zh/api/#routelocationraw): 通过一个路由地址跳转到一个不同的地址，就像你调用 [router.push()](https://router.vuejs.org/zh/api/#push)

```javascript
const router = createRouter({ ... })

// 每个守卫方法接收两个参数：
// to: 即将要进入的目标
// from: 当前导航正要离开的路由
router.beforeEach((to, from) => {
  // ...
  // 返回 false 以取消导航
  // return false
  // 或者将用户重定向到登录页面 等价于 return "/login"
  return { name: 'Login' }
})
```
 Vue2
```javascript
router.beforeEach((to,from,next)=>{
	//...
  next() //跳转
})
```
## 全局解析守卫
和 router.beforeEach 类似，区别是在导航被确认之前，同时在所有组件内守卫和异步路由组件被解析之后，解析守卫就被调用。 
```javascript
router.beforeResolve((to,from)=>{})
```
## 全局后置钩子 
```
router.afterEach((to, from) => {})
```

## 路由独享的守卫（1个）
在路由配置上直接定义 beforeEnter 守卫  (只有一个)
```javascript
{
  path: '/foo',
    component: Foo,
    beforeEnter: (to, from) => {
    // ...
  }
}
```
## 组件内的守卫 （3个）
```javascript
<script>
export default {
    beforeRouteEnter: (to, from, next) => {
        console.log('beforeRouteEnter');
        // 在渲染该组件的对应路由被验证前调用
        // 不能获取组件实例 `this` ！
        // 因为当守卫执行时，组件实例还没被创建！
        next(vm => {
            // 通过 `vm` 访问组件实例
        })
    },
    beforeRouteLeave: ((to, from) => {
        console.log('onBeforeRouteLeave');
        // 取消导航并停留在同一页面上
        //return false
    }),
    beforeRouteUpdate: ((to, from) => {
        console.log('onBeforeRouteUpdate');

        // 取消导航并停留在同一页面上
        //return false
    })
}
</script>
```
 script setup只支持 onBeforeRouteLeave,  onBeforeRouteUpdate
```javascript
<script setup>
import { onBeforeRouteLeave, onBeforeRouteUpdate } from 'vue-router'
onBeforeRouteLeave((to, from) => {
    console.log('onBeforeRouteLeave');
    // 取消导航并停留在同一页面上
    //return false
})
onBeforeRouteUpdate((to, from) => {
    console.log('onBeforeRouteUpdate');

    // 取消导航并停留在同一页面上
    //return false
})
</script>
```
#  7. 面试题： 完整的导航解析流程

1. 导航被触发。

2. 在失活的组件里调用 `beforeRouteLeave` 守卫。

3. 调用全局的 `beforeEach` 守卫。

4. 在重用的组件里调用 `beforeRouteUpdate` 守卫 (2.2+)。  

      （针对动态路由组件，路由地址没有变化，只是参数发生变化时触发）

5. 在路由配置里调用 `beforeEnter`。（路由独享守卫）

6. 解析异步路由组件。

7. 在被激活的组件里调用 `beforeRouteEnter`。 （组件内守卫）

8. 调用全局的 `beforeResolve` 守卫 (2.5+)。

9. 导航被确认。

10. 调用全局的 `afterEach` 钩子。

11. 触发 DOM 更新。

12. 调用 `beforeRouteEnter` 守卫中传给 `next` 的回调函数，创建好的组件实例会作为回调函数的参数传入。
# 8. route和router的区别（重要）
## Vue3
在一个vue组件中（普通组件和路由组件都能访问到），都可以访问到这两个对象，它俩的区别是
```javascript
const route = useRoute()
const router = useRouter()
```

- route是路由记录(信息)对象，只读，存储了与路由相关的信息 
```javascript
fullPath: "/index/recommend"
matched: (2) [{…}, {…}]    //存储了所有级别路由的信息  （如果有两个，则matched[0]是一级路由信息，matched[1]是嵌套路由信息）
meta: {title: '首页-推荐'}
name: "recommend"  //在路由配置中给路由命的名字
params: {}
path: "/index/recommend"
query: {}
```
  router是路由对象，可以用来进行路由跳转

```js
router.push()
router.replace()
router.go()
```



## Vue2
```javascript
this.$route.params.XX
this.$router.push()
```

# 11. 路由懒加载和项目打包
当打包构建应用时，JavaScript 包会变得非常大，影响页面加载。把不同路由对应的组件分割成不同的代码块，然后当路由被访问的时候才加载对应组件，这样就更加高效了。

- 实现方式 
```javascript
// 将
// import UserDetails from './views/UserDetails.vue'
// 替换成
const UserDetails = () => import('./views/UserDetails.vue')
// const UserDetails = () =>
//   import(/* webpackChunkName: "group-user" */ './UserDetails.vue')

const router = createRouter({
  // ...
  routes: [{ path: '/users/:id', component: UserDetails }],
})
```

- 项目打包 
```
npm run build
```
# 12. 404页面的配置
如果用户访问的路由在路由表中没有匹配路径，则应该显示404页面
实现方式： 在所有路由表的最后，添加以下面的配置

## Vue3
```javascript
...
import NotFound from "../views/NotFound.vue";

const routes = [
   ...
    { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound },
];

```
## Vue2
```javascript
{
  // *表示所有路由，如果在上面没有匹配到，就会执行该路由，显示404页面
  // 这个配置必须放在所有路由的最下面
  path: '*',
    component: ()=> import ('@/views/NotFound.vue'),
}
```
# 13. 动态添加路由
router.addRoute() 动态添加路由
Vue Router 提供了两个功能来查看现有的路由：

- [router.hasRoute()](https://router.vuejs.org/zh/api/interfaces/Router.html#Methods-hasRoute)：检查路由是否存在。(参数必须是路由名称)
- [router.getRoutes()](https://router.vuejs.org/zh/api/interfaces/Router.html#Methods-getRoutes)：获取一个包含所有路由记录的数组。
```javascript
router.addRoute({ name: 'admin', path: '/admin', component: Admin })
//添加嵌套路由
router.addRoute('admin', { path: 'settings', component: AdminSettings })
```
这等效于：
**js**
```javascript
router.addRoute({
  name: 'admin',
  path: '/admin',
  component: Admin,
  children: [{ path: 'settings', component: AdminSettings }],
})
```
