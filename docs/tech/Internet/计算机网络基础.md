## 2. **网络诊断与排错**
- **所属领域**: 系统运维 / 网络管理

- **核心技能**:
  - 网络连通性测试
  
  ```
  # 1. 查看所有网络接口
  ifconfig
  ifconfig en0 | grep "ether"   查看mac地址
  ifconfig en0 | grep "inet"    查看ip地址
  ifconfig en0 | grep status    检查网络接口状态
  # 2. 查看默认网关
  netstat -nr | grep default
  
  # 3. 测试本地网络栈
  ping 127.0.0.1
  
  # 4. 测试到网关（先用上一步找到网关地址）
  ping 192.168.1.1
  
  # 5. 测试互联网连通性（绕过DNS）
  ping 8.8.8.8
  
  # 6. 测试DNS解析
  ping www.apple.com
  
  # 7. 如果DNS有问题，使用特定DNS服务器测试
  nslookup www.apple.com 8.8.8.8
  
  # 8. 追踪到目标网站的路径
  traceroute www.google.com
  ```
  
  - 端口和服务检测
  
  ```
  # 测试80端口 (HTTP)
  telnet 5edata.com 80
  
  # 测试443端口 (HTTPS)  
  telnet 5edata.com 443
  
  # 测试22端口 (SSH)
  telnet 5edata.com 22
  ```

## 3. **具体技术分层**

### 网络层 (ping)
```
知识领域: 网络协议基础
相关概念: 
  - ICMP 协议
# 基本用法  
ping 8.8.8.8  （测试ip地址是否可以抵达）
traceroute google.com （显示数据包到达目标经过的所有路由节点）
  - IP 路由
  - 子网划分
  - 网关和DNS
应用场景: 网络工程师、运维工程师
```

### 传输层 (telnet)
```
知识领域: 网络编程基础  
相关概念:
  - TCP
# 需要可靠传输的应用
🌐 网页浏览 (HTTP/HTTPS)
📧 电子邮件 (SMTP/POP3/IMAP)
💾 文件传输 (FTP/SFTP)
🛡️ 远程登录 (SSH/Telnet)
🗄️ 数据库连接 (MySQL/PostgreSQL)
UDP 协议
# 实时性要求高的应用
🎥 视频流媒体 (Zoom/直播)
🎮 在线游戏 (实时对战)
📞 语音通话 (VoIP)
🔍 域名解析 (DNS)
📡 网络时间同步 (NTP)
  - 端口和Socket
#查看所有监听端口
netstat -tuln
# 端口就像大楼里的房间号
IP地址 = 大楼地址 (192.168.1.100)
端口号 = 房间号 (80, 443, 22等)
# 周知端口 (0-1023) - 系统服务
80  → HTTP     (网页服务)
443 → HTTPS    (安全网页)
22  → SSH      (安全远程登录)
21  → FTP      (文件传输)
53  → DNS      (域名解析)
# 注册端口 (1024-49151) - 应用程序
3306 → MySQL   (数据库)
3389 → RDP     (远程桌面)
5432 → PostgreSQL
# 动态端口 (49152-65535) - 客户端临时使用
# 数据包通过端口找到正确的应用程序
📦 数据包 → IP地址:端口 → 🖥️ 具体应用
# 查看端口占用
sudo lsof -i :端口号
# Socket = IP地址 + 端口号
# 它是网络通信的端点
Socket A: 192.168.1.100:8080  (服务器监听)
Socket B: 192.168.1.50:54321   (客户端连接)
  - 三次握手
# TCP 就像可靠的快递服务：
✅ 保证送达（确认机制）
✅ 按顺序送达（序列号）
✅ 知道包裹大小（数据分段）
✅ 适应路况（拥塞控制）
第一次：客户端说"我想连接"
第二次：服务器说"我准备好了，你准备好了吗"
第三次：客户端说"我也准备好了"
确保双方都能正常收发数据

  - 连接状态
# 查看所有连接状态
netstat -an | grep tcp
（大量TIME_WAIT）
（大量CLOSE_WAIT）
（SYN_RCVD与 SYN Flood攻击）
应用场景: 后端开发、运维、安全测试
```
