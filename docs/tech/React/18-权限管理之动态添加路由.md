## 一.路由权限管理的流程

### 1. 定义角色

根据项目需求，可在项目中实现角色管理 

管理员拥有更高级的权限，普通用户只能访问部分页面或执行特定操作。具体的权限定义可以在后端或前端进行管理，在数据库或配置文件中存储角色和权限的映射关系。

比如： 教培行业的项目

1. 超级管理员（Super Admin）: 超级管理员是系统的最高权限角色，拥有对系统的完全控制权。他们的职责包括管理和配置整个系统的功能、权限和设置，创建和管理其他管理员账号，监控系统的运行状况和安全性，处理系统级别的问题和决策。
2. 机构管理员（Institution Admin）: 机构管理员是教育机构或培训机构内部的管理员角色。他们管理和运营特定机构的教育资源、课程设置和学生信息。其职责包括创建和管理教师账号、设置教师权限、创建和管理课程、课程安排和学生管理、监督教学质量和教务运营等。
3. 教师（Teacher）: 教师是教育机构中的教学人员，在后台项目中具有特定的权限和功能。他们可以创建和管理自己的课程、布置作业、评估学生作业和表现，与学生和家长进行沟通，查看和管理学生的学习进度和成绩等。
4. 学生（Student）: 学生是教育机构中的学习者，他们在后台项目中可以查看自己的课程内容、作业要求和成绩，与教师和家长进行交流，获取学习资源和支持等。
5. 家长（Parent）: 家长在教育机构中扮演重要的角色，他们可以通过后台项目查看和监控自己孩子的学习进展、作业完成情况、学习资源和成绩报告，与教师进行沟通和讨论，参与学校或教育机构的相关事务。

### login.jsx中核心代码

```jsx

//登录成功后，把后端返回的路由数据存入sessionStorage实现持久化
// 还要存入store, 提供给路由配置和导航菜单这些功能

//-----------------------------------------------登录成功
  const onLoginSuccess = (token, values,routesData) => {
    sessionStorage.setItem("token", token);
    //登录成功后，把后端返回的路由数据存入sessionStorage实现持久化
    sessionStorage.setItem("routesData", JSON.stringify(routesData));
      
    // 还要存入store, 提供给路由配置和导航菜单这些功能
    dispatch(addRoutesData(routesData))
  };

//表单数据验证通过
  const onFinish = (values) => {
    setTimeout(() => {
      //后端接口根据用户角色返回的数据
      let res = {
        code: 0,
        token: '111111111111111',
        routesData  // 即为上面路由数据示例中的json
      }
      let { code, token } = res;
        
      if (code === 0) {
        onLoginSuccess(token, values,routesData);
      }
    }, 1000)
  };
```

```jsx
export default function AdminMenu() {
 
  //从store中拿到路由数据
  const {routesData} = useSelector(state=>state.tabs)

  // 初始化处理路由数据,
  
  useEffect(() => {
     let list = routesData.map(subMenu=>{
        let childrenList = []
        if (subMenu.children){
          childrenList = subMenu.children.map(menu=>{
            return {
              ...menu,
              icon: <Iconfont icon={menu.icon}></Iconfont>
```

```jsx
  useEffect(() => {
```
