# 1. 获取元素常见的尺寸

我们在网页上通常要获取到元素的尺寸，可以从样式入手，用getComputedStyle获取。但是因为带的有单位，并且获取时不方便。所以我们可以使用其他操作

## 常见尺寸获取

### clientWidth / clientHeight

元素padding + width / padding + height

### offsetWidth / offsetHeight

获取元素的border + padding + width / border + padding + height

能够最精确的获取到元素所占的空间

> 注意结果和实际不符，差1px, 是因为win10系统， 在"设置"-"屏幕"-"缩放"中设置为推荐比例即可

### offsetLeft / offsetTop

元素距离其offsetParent的边框内部的距离

## offsetParent

绝对定位元素的祖先元素中，最近哪个元素有定位属性，这个元素就是它的offsetParent

### scrollWidth / scrollHeight

整体内容高度包含padding值，不包含border，内容溢出才会有所改变，不溢出，则和clientWidth / clientHeight一样。前者不常用。

document.documentElement.clientHeight 窗口高度

document.documentElement.scrollHeight 页面高度（没有滚动条时和窗口高度相同，有滚动条时大于窗口高度）

### scrollLeft / scrollTop 可读可写

- element.scrollTop 元素滚动条偏移值

- document.documentElement.scrollTop : 窗口滚动条的偏移值

## display:none 和 visibility:hidden的区别

前者消失，无法用js获取其尺寸，后者不可见，还占位，但不会遮盖其他元素。js可以正常获取对应的尺寸。

## getBoundingClientRect()方法

- Element.getBoundingClientRect() 方法返回元素的大小及其相对于视口的位置

  拥有left, top, right, bottom, x, y, width, 和 height这几个以像素为单位的只读属性用于描述整个边框。除了width 和 height 以外的属性是相对于视图窗口的左上角来计算的。

  由于兼容性问题，尽量仅使用 left, top, right, 和 bottom.属性是最安全的

- document.documentElement.getBoundingClientRect() 方法返回视口相关的数据

# 2. 楼层导航案例中涉及的知识点

## 2.1 如何把伪数组转换为真数组

```js
let contentList = document.querySelectorAll('.main .content')
//报错，因为contentList是伪数组，不能使用map方法
let arr = contentList.map()
```

解决方案： 使用ES6语法的展开运算符 "..."

```js
let contentList = document.querySelectorAll('.main .content')
// 等价于把contentList中的每个dom对象取出来，存入一个新数组中   [dom1,dom2,dom3,dom4]
listHeight = [...contentList].map((item) => item.offsetTop)
```

## 2.2 如何给同一个事件绑定多个事件处理程序

注意： 如果用addEventListener给同一dom相同事件多次绑定相同的处理程序，则事件发生时，事件处理程序会执行多次。

```js
//只能执行最后一个
dom.onclick = function () {}
dom.onclick = function () {}
```

```js
// 可以执行所有
window.addEventListener('scroll', function () {
  console.log(1)
})
window.addEventListener('scroll', function () {
  console.log(2)
})
```

## 2.3 offsetTop 在循环中带来的性能问题

在一个循环中反复取得一个元素的*offsetTop*值,事实上,在此代码中该元素的offsetTop值并不会变更,所以会存在不必要*的性能*损耗

解决方案：

提前在循环外部获取offetTop的值，不要把获取的过程放在循环内部

## 2.5 单击导航切换时，滑块会出现卡顿

参考案例：97-js-楼层导航-用定时器改造滚动条的滚动效果.html

- 分析原因：单击导航事件和页面滚动事件都要控制滑块的位置，单击导航事件时，同时触发了页面滚动，所以两个逻辑同时对滑块做处理产生了冲突

- 解决方案：

  单击导航时，把页面滚动事件解除绑定，在导航触发的页面滚动结束后，再重新绑定滚动事件的处理

## 2.6 单独用js定时器实现页面的滚动时的bug

- 描述问题

  正常触发页面滚动后，页面会滚动到指定的位置 ，但是如果两次触发的间隔较小，页面就会上下抖动

- 分析原因

  原本触发滚动后，滚动条到达目标值，定时器会清除。但是快速触发多次滚动，就会在上一次定时器没有清除的情况下又触发了另一个定时器，两个定时器都在运行，控制滚动条的位置 ，所以引发了冲突

- 解决方案：

  在任何时候开启一个新的定时器之前，一定要先清除原先的定时器，这样保证只有一个定时器在触发

## 2.7 作用域引发的问题

- 描述问题

  当用js定时器实现楼层导航的页面滚动时，在封装的setScroll函数内部 用let声明了timer, 会发现快速切换导航时，同样会引起页面滚动时上下抖动的效果，但是在setScroll函数中已经清除了定时器，按说不应该出现问题

- 分析原因

  因为 let timer声明在setScroll函数内部，是函数内部作用域，而快速切换导航时，触发的第二次setScroll中的清除定时器的功能并不能清除第一次setScroll中触发的定时器，也就是多个定时器的冲突

- 解决方案

  提升timer的作用域，把let timer声明放在setScroll的外面即可

# 3. js交互效果中如何处理rem单位

1. 如果基于原来用rem设计的元素上来获取对应的px的值，是不需要做额外处理，直接得到px并用在相应元素上即可

​ 2. 如果是通过js直接给元素设置rem尺寸，则把设计稿中的px直接换算为rem,进行设置即可

​ 3. 如果变量需要设置为rem来实现适配，但在计算中又需要rem转换为px

​ 1. 获取html的fontSize的值

​ 2. 用rem的值 \* fontSize的值得到px
