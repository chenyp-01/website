# js中的事件

DOM中的事件是一个很重要的东西，它可以让用户和浏览器之间进行交互，以此来实现人机交互效果。

## DOM事件

DOM事件分为DOM0级事件和DOM2级事件。DOM0级其实不存在，我们把DOM最初的版本叫0级。在DOM2级的时候更新了一种新的事件绑定的方式，所以才有了所谓DOM0级和DOM2级事件。

DOM事件在过去使用的过程中，语法很简单。

## 绑定事件


DOM0级事件绑定：

```js
DOM.on事件名 = function () {
  
}

// 该方法绑定时无法同时给同一个元素绑定多个相同的事件
DOM.onclick = function () {}
DOM.onclick = function () {} // 这个会覆盖上面的
```

DOM2级事件绑定:

```js
//isCapture在讲事件冒泡的环节再介绍
// isCapture是布尔值，默认值是false
DOM.addEventListener("事件名", 事件句柄, isCapture)  

// 一般这么写
DOM.addEventListener("事件名", function () {}, false)
```

## 移除事件

DOM0级事件移除

```js
DOM.onclick = null
```

DOM2级事件移除

```js
DOM.removeEventListener("事件名", 事件句柄, isCapture)
```

举例

```js
function handler () {}

DOM.addEventListener('事件名', handler, false)
DOM.removeEventListener('事件名', handler, false)

DOM.addEventListener('事件名', handler, true)
DOM.removeEventListener('事件名', handler, true)
```

## 移动端点击事件延迟

- 延迟原因

1. 移动端浏览器在派发点击事件的时候，通常会出现300ms左右的延迟。
2. 原因: 移动端的双击会缩放导致click判断延迟。
   这是为了检查用户是否在做双击。为了能够立即响应用户的点击事件，才有了FastClick

- 解决方案： 简单了解

  参考 ： <https://www.likecs.com/show-307492379.html>

- 常用解决方案： 引入fastclick.js

  ```js
  <script  src='./js/fastclick.js'></script>
  ```

  ```js
  if ('addEventListener' in document) {
      document.addEventListener('DOMContentLoaded', function() {
          FastClick.attach(document.body);
      }, false);
  }
  ```

## 事件流

事件流决定在哪个阶段执行事件

### DOM2级事件标准事件流 

在标准的事件流中，先捕获，到达事件源(目标)元素，再冒泡。具体是在捕获阶段执行事件还是在冒泡阶段执行事件，需要看第三个参数是true还是false

```js
// *******第三个参数默认值为false，一般用事件冒泡
// 在冒泡阶段执行事件
DOM.addEventListener("事件", function () {}, false) 
// 在捕获阶段执行事件
DOM.addEventListener("事件", function () {}, true) 
```

### 什么是事件冒泡 (面试题)

在标准的事件流中，当我们触发一个元素的事件时，会经历三个阶段：

1.  捕获阶段，从document沿着DOM树向下捕获， 直到找到这个目标元素为止，
2.  目标阶段： 如果目标元素绑定的有该事件，则执行
3. 冒泡阶段： 之后从这个元素沿着DOM树向上冒泡，有相同事件的元素会执行对应元素上的事件，没有则继续冒泡，直到到document结束 

### 事件委托（事件代理） ---（面试题）

事件代理”即是把原本需要绑定在子元素的响应事件委托给父元素或祖先元素，让父元素或祖先元素担当事件监听的职务。事件代理即是利用事件冒泡的机制把里层所需要响应的事件绑定到外层

事件委托的优点

- 可以大量节省内存占用，减少事件注册
- 可以实现当新增子对象时无需再次对其绑定

```js
祖先元素.on事件类型 = function (e) {
  if (e.target.xxx === "xxx") {
    // 执行对应的操作
  }
}
```

### 阻止冒泡 

冒泡在有些时候需要被阻止，当元素本身和内部元素有相同类型的事件执行时，一般需要阻止冒泡

```js
dom.onclick = function (e) {
  //阻止冒泡
  e.stopPropagation()
}
```

## 事件对象

事件对象是在所有的事件中都会自动生成的一个对象。这个对象叫事件对象Event

### 如何获取到事件对象

直接在事件函数中，填入形参e即可在函数中把e当成事件对象使用。

```js
DOM.on事件名 = function (e) {
  // 给事件函数写一个形参一般用 e evt event 都可以
  // 在函数内部就可以使用这个参数了
  console.log(e) // 千万不要考虑，这个e哪来的。
}

DOM.addEventListener("事件名", function (e) {}, false)
```

### 阻止浏览器默认行为

诸如a标签跳转，表单提交，鼠标右键打开菜单，这些操作都是浏览器的默认行为。

a标签跳转，在网页中，有很多时候，我们需要借助a标签的特性，但是又不想发生任何跳转，我们就需要取消浏览器默认行为

```js
dom.on事件 = function (e) {
  e.preventDefault()
}
```

还可以直接返回false

```js
dom.on事件 = function () {
  // 其他代码
  
  return false
}
```

一般a标签不会用这个方式，我们会使用

```js
<a href="javascript:;"></a>
<a href="javascript:void(0);"></a>
```

### 事件对象中的位置信息

 e.screenX / e.screenY 

鼠标点击的位置距离屏幕左侧/顶部的距离

 e.clientX / e.clientY   

鼠标点击的位置，距离浏览器窗口左侧/顶部的距离

 e.layerX / e.layerY 

默认情况下，和clientX一样，如果元素中有定位属性，则基于该元素offsetParent的距离，如果元素自身有定位属性，则距离自身左侧/顶部的距离。

 e.offsetX / e.offsetY 

距离border内侧的距离

e.keyCode / e.key

获取按键

```js
document.onkeyup = function(e){
    console.log(e.keyCode);  // ASCII码
    console.log(e.key);  //键名
}

```

e.target

触发事件的目标元素

e.currentTarget

绑定事件的目标元素
