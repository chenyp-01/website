## 正则表达式

### 1. 正则表达式

英文： Regular Expression

- 正则表达式是具有特殊语法的字符串，用来表示指定字符或者字符串在另一个字符串中出现的情况。
- 作用：正则表达式可用于所有**文本搜索和文本替换**的操作。

### 2. 正则表达式的定义

#### 2.1 基于RegExp 对象的构造函数写法

语法：new RegExp(pattern,attributes)

- pattern:是一个字符串，指定了正则表达式的模式
- attributes:修饰符，用以说明高级匹配模式的规则（可选参数，并且可以组合使用）
  g 执行一个全局匹配，简而言之，即找到所有的匹配，而不是在找到第一个之后就停止（global）
  i 执行不区分大小写的匹配 （ignore）

```js
var str = 'welcome';
var re = new RegExp('a','i');//只会匹配字符串中出现的第一个字母a(不区分大小写)
// test检索字符串中是否有指定的值。返回 true 或 false
console.log(re.test(str));

```

#### 2.2 字面量写法

语法：/pattern/attribute

>  参数: 参考创建RegExp对象里面的参

```js
var re;
re = /a/; //只会匹配字符串中出现的第一个小写字母a
re = /a/g; //匹配字符串中出现的所有小写字母a
re = /a/i; //只会匹配字符串中出现的第一个大写A或者小写a
re = /a/ig; //只匹配字符串中出现的所有大写A或者小写a 
```

> 正则构造函数和字面量的区别： 在构造函数生成的正则中， 转义字符\ 要写两个， 而字面量中是一个

###　3. 正则表达式的执行

#### 3.1 正则表达式对象的方法
- test方法

  检索字符串中是否有指定的值。返回 true 或 false。（用的最多）

  ```js
  // 语法： 正则.test( 要匹配的字符串 )
  let patt1 = /aBc/i //表示包含'aBc'字符串，并且忽略大小写
  let patt2 = new RegExp("ac","i")  //表示包含'ac'字符串，并且忽略大小写
  let str = 'ddfabcfjdkabc'
  let result1 = patt1.test(str)
  let result2 = patt2.test(str)
  console.log('result1',result1); //true
  console.log('result2',result2); //false
  ```

#### 3.2 String 对象中支持正则表达式的方法

- string.match()	

  找到一个或多个正则表达式的匹配（用的最多）

  ```js
  // match方法: 找到一个或多个正则表达式的匹配
  
  // 非全局匹配：  只找到符合条件的第一个
  // 全局匹配： 查找到所有符合条件的记录，并存入数组
  
  // 查找所有"a数字b"的字符串，统计个数
  let user = 'a1baaa2b    a6b';
  // //查找到符合条件的第一个子字符串
  let result = user.match(/a[0-9]b/g)  
  console.log(result);
  console.log(result[0]);
  console.log(result.length);
  ```

  > 讲完分组后可以再来看match最完整的用法
  >
  > 1. 如果正则不是全局匹配，并且没有分组(), 那么结果为数组，数组中只有一个元素，即正则匹配到的字符串
  >
  > ​    2. 如果正则不是全局匹配，但是有分组(),那么结果为数组，数组中第一个元素是整个正则匹配的部分，而数组中其它元素是反向引用的字符串
  >
  > ​    3. 如果正则是全局匹配，则结果为数组，存储所有正则匹配到的字符串，但不再存储反向引用的内容

- string.replace()

  替换与正则表达式匹配的子串（用的最多） 
  注意：字符串的replace方法的第一个参数可以传入普通字符串还可以传入正则表达式

  ```js
  // replace方法 : 替换与正则表达式匹配的子串
  var user = 'i am Alice is my friealicend,Alice ... Alice,alice';
  
  var result = user.replace(/Alice/,'Rose')
  // var result = user.replace(/Alice/g,'Rose')
  // var result = user.replace(/Alice/gi,'Rose')
  
  console.log(result);
  ```

### 4. 正则表达式语法

#### 宽度

正则表达式中一个字符是一个宽度，会和字符串中的对应的符合条件的字符尽心匹配，如： 

```js
let reg = /a/
let str = "sva"
// reg就会匹配到a

let reg = /aa/
let str = 'svab'
// 这个不能匹配
```

#### 如果我们要匹配的内容是一个范围

使用`[]`表示一个范围，宽度为1

```js
let reg = /[abcdefghjklmnopqrstuvwxyz]/ // 虽然有26个字母，但是只匹配一个宽度
let reg = /[a-z]/ // 如果是连贯的字母，就可以写a-xxx 如a-c表示 abc
let reg = /[a-zA-Z]/
let reg = /[a-cA-C_0-4]/
```

####　如果表示不包含[]中的内容

```js
/[^a-c]/ // 不包含a-c 
```

#### 以什么开始 以什么结束

为了确保我们匹配到的字符是一个完整没有其他不符合要求的字符，一般我们需要设置开始和结束的标志

```js
let reg = /^规则/
let reg = /规则$/
let reg = /^规则$/
```

#### 将匹配宽度增加

在我们进行匹配时，有很多时候会出现相同匹配情况，我们可以设置匹配宽度 {m, n}例如：

QQ号是5-11位

```js
let qqReg = /^[1-9][0-9]{4,10}$/
```

数量可以有以下写法

```js
{m, n} // m - n位
{m} // 固定m位
{m,} // 大于等于m位
```

#### \ 转义符

我们正则表达式中，有些字符有自己的功能，我们就需要通过转义符来实现另外一种功能，例如

- d表示字母d，\d表示数字
- w表示字母w，\w表示 字母，数字，下划线
- \表示转义，\\\表示普通的符号\

#### 字符大写

如果一个元字符中字母大写，则一般表示小写的反义结果，例如

\d表示数字 \D 表示非数字

#### * + ?

- \* 表示 0次或者多次 {0, }
- +表示1次或者多次 {1, }
- ? 表示0次或者1次 {0, 1}

#### . 点字符

.是除换行符外的任意字符

如果我们想要匹配标点符号`.`我们需要`\.`

#### \b \B

表示单词的边界，一般你用不上，例如，我想匹配我的className

```js
<div class="box box1 box2"></div>
```

\B表示非单词边界

#### \d \D

\d等同于[0-9] 表示数字

\D等同于[^0-9]  非数字

#### \n \s \S \r

\n换行符 \r回车符  \t制表符

\s空白符   \n \r \t \v都是空白符， 空格也是空白符

> 了解一下
>
> \v 代表垂直制表符(Vertical Tab)。它是一种控制字符,用于控制打印机将打印头移动到下一个垂直制表位。在打印输出和文本显示中,垂直制表符会使打印机或者显示器将文本输出位置移动到下一个垂直制表位。

\S表示非空白符



#### \w \W

\w字母数字下划线 等同于 [a-zA-Z0-9_]

\W表示非数字字母下划线[^a-zA-Z0-9_]

#### \数字

我们如果在正则表达式中有括号，会产生一个分组

\1表示引用正则中第一个左括号的内容, 其实访问的就是$1中存储的字符串

```js
let reg = /(\d)\1/g // 等同于 \d前面匹配内容的引用
let str = '112234' // 11 22

let reg = /(\d)\1([a-z])\d\2/g; // 等同于 \d前面匹配内容的引用
let str = "fffffff33a1a---22b4bgggggg"; // 11 22
let result = str.match(reg)
console.log(result)
```

#### ()

优先匹配 产生分组

```js
var user= "redredred";
var patt = /red{3}/;
var patt = /(red){3}/;
var result = patt.test(user)
console.log(result);

```

#### | 或

条件，不走前面就走后面

```js
let reg = /(13\d)|(13[a-c])/;
// let reg = /^(131)|(132)/
let str = "13b33556677";
let result = reg.test(str);
console.log(result)
console.log(RegExp.$1); // 13b
```

### 贪婪和懒惰

如果在限制符后添加`?`会进入懒惰模式，然后会尽可能少的取匹配对应的字符。默认是贪婪模式，尽可能多的匹配字符。

```js
// 默认的贪婪匹配是从后往前匹配，最大长度的匹配，惰性匹配就是在量词后面加个？从字符串的前面开始匹配，最小长度的匹配

// 贪婪模式
var result = '12345678901233'.match(/\d{3,5}/g);   //默认的匹配方式是贪婪模式
console.log(result);

// 惰性模式:  在量词后加上 "?" 即可,: 在满足条件的前提下，尽可能少的匹配字符

var result = '12345678901'.match(/\d{3,5}?/g);
console.log(result);
```

### 非捕获组 (?:pattern)

普通的括号组(())可以将模式中的一部分隔离出来,并且这个组的匹配结果也会被捕获,可以在替换字符串中通过 $1 来引用。但是有时候,我们只是想将一部分模式隔离出来,但并不需要捕获这个组的匹配结果。这时就可以使用非捕获组(?:pattern)。

```js
let str = `
    windows12
    windows11
`
// 相当于 /windows10|windows11/ ,匹配到的是windows10 和 windows11
let reg = /windows(?:10|11)/
let result = reg.test(str)
console.log(result);
```

### 零宽断言

####  (?=pattern) 

零宽正预测先行断言:断定接下来会匹配的内容

```js
 <script>
      let str = `
	w11s10
  w22s11
  w33sxp
`;
      let reg = /w\d{2}s(?=\d+)/g;
      console.log(str.match(reg))
```
#### (?!pattern)

和第一个都是后面判断，表示不符合条件

```js
let str = `
	windows10
  windows11
  windowsxp
`
let reg = /windows(?!\d+)/
// 匹配到的是windows 是windowsxp的windows 前面两个比匹配
console.log(str.match(reg))
```
#### (?<=pattern)

零宽正回顾后发断言:断定前面已经匹配的内容

```js
let str = `
    windows12
    windows11
`
// 为了匹配\d+,但只能是windows字符串后面的数字才行
let reg =  /(?<=windows)\d+/g
let result = str.match(reg)  //['12', '11']
console.log(result);
```



#### (?<!pattern)

和上面方向相反，先判断后匹配

```js
let str = `
	windows1
  windows2
  ubuntu3
`
let reg = /(?<!windows)\d+/
// 匹配到的是后面的数字，但是是最后一个3，因为前面不能是windows
```
