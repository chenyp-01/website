# 2. 函数重点总结

## 2.1 使用函数的意义

- 复用代码

- 封装代码

- 使代码更为清晰，便于后期维护

## 2.2 函数定义的两种方式

函数声明和函数表达式最重要的区别是: 函数声明可以在定义之前调用 ，而函数表达式不能在定义之前调用

1. 函数声明

   ```js
   function 函数名(参数) {
     代码块
   }
   ```

2. 函数表达式

   ```js
    function(参数){代码块}
   ```

## 2.3 函数的参数

- 函数定义中的参数称为形参

- 函数调用中的参数称为实参

- 特殊情况：

  1.实参个数大于形参

  ​ 在函数内部通过arguments可以得到全部实参

  2.实参个数小于形参

  ​ 没有接收到实参的形参，值为undefined

## 2.4 函数作用域

全局变量(全局作用域)和局部变量（局部作用域）

- 局部变量： 局部变量只能在当前函数内部被访问，函数外部不能访问

  ​ 1. 函数的形参

  ​ 2. 在函数内部用var声明和用function声明的变量

- 全局变量： 在整个代码中任意一个位置 都可访问 (存和取)

  ​ 除了局部变量，其它都是全局变量

  ​ 1. 在函数外部用var声明或用function声明的变量

      2. 在整个脚本中没有用var声明但是直接使用的变量 (尽可能要避免)

## 2.5 函数的返回值

从函数是否有返回值来区分，函数分为两种

- 没有返回值的函数，即函数内部没有return语句

  ```js
  // 注意： 没有返回值的函数直接调用即可
  function fun1() {
    var num1 = 9,
      num2 = 23
    console.log(num1 + num2)
  }
  fun1()
  ```

- 有返回值的函数，即函数内部有return语句

  ```js
  // 注意： 有返回值的函数，如果return后有值，则一定会和赋值或输出结合在一起
  // 函数内部，return之后的代码都不会被执行，也就是说return会中止函数运行
  // return之后如果没有值，仅仅表示跳出函数

  function fun2() {
    var num1 = 9,
      num2 = 23
    var result = num1 + num2
    return result
  }
  var value = fun2() //32
  console.log(fun()) //32
  ```

  ```js

      return item + '件'
    }
  })

  ```

- array.every()

  ```js
  //every一定有返回值，返回值一定是布尔值， true表示全部及格，false表示有人不及格

  var result = arr.every(function (item, index) {
  ```


### 4. 重要算法的理解---理解并能够手写

- 冒泡排序

  规则：从左到右，数组中相邻的两个元素进行比较，将较大的放到后面。

  规律：冒泡排序每一轮排序，都可以找出一个较大的数，放在正确的位置。

  比较的轮数 = 数组长度 - 1;

  每一轮比较的次数 = 数组长度 - 当前的轮数

  ```js
  var score = [89, 34, 56, 67, 90, 23, 68, 100, 85, 12]
  for (var j = 0; j < score.length - 1; j++) {
    // j表示第几轮
    console.log('第' + (j + 1) + '轮开始')
    for (var i = 0; i < score.length - (j + 1); i++) {
      if (score[i] > score[i + 1]) {
        var temp = score[i]
        score[i] = score[i + 1]
        score[i + 1] = temp
      }
      console.log(score) // [34,89,56,67,90,23]
    }
    console.log('第' + (j + 1) + '轮的结果是', score)
  }
  ```

- 数组去重

  ```js
  var arr = [23, 23, 45, 23, 56, 45, 89, 100, 78]
  // 需求： 去除arr中所有的重复元素
  // arr =  [23,45,56,89,100,78]
  var newArr = [] // [23]
  var flag

  for (var n = 0; n < arr.length; n++) {
    flag = false //false表示没有重复, true表示有重复
    for (var i = 0; i < newArr.length; i++) {
      if (arr[n] === newArr[i]) {
        flag = true
        break
      }
    }
    if (!flag) newArr.push(arr[n])
  }
  console.log(newArr)
  ```

## 2. 浮点运算精度丢失的问题

浮点数就是有小数位的数值

```js
0.1 + 0.2 // 本来结果是0.3 实际 0.30000000000000004
```

- 原因

  浮点数运算时会造成精度丢失问题，原因是十进制的浮点数在运算时要先转为二进制，而二进制就造成精度丢失

- 解决方案

  方案一： 先把浮点数乘于10的n次方，转为整数， 计算完毕后再除以10的n次方

  ```js
  //直接处理浮点数的运算，结果精度丢失
  var num1 = 6.8 - 5.6
  console.log(num1) // 1.2000000000000002
  var num2 = 1.2
  console.log(num1 == num2) // false

  // 解决方案
  var num1 = (6.8 * 10 - 5.6 * 10) / 10
  console.log(num1) // 1.2
  console.log(num1 === 1.2) //true
  ```
