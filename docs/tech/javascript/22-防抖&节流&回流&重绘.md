# 1. 防抖和节流（高频面试题）

## 1.1 应用场景

- 防抖(debounce)和节流(throttle)都是用于控制函数调用频率的技术。它们都有助于优化页面性能,改善用户体验。

- 应用场景

  window.onscroll(无限滚动加载，楼层导航)

  window.onresize

  dom.onmousemove(拖拽效果)

  input.onkeyup(搜索框的按键)

  以上场景由于事件频繁被触发，而重复执行DOM操作、资源加载等重行为、会导致UI停顿甚至浏览器崩溃。

  这些事件需要防抖或节流

- 案例

  ```js
  <div id="box">
      hello world
  </div>

  <script>

      function fn(){
          console.log('业务逻辑');
      }
      // window.onresize= function(){
      //     fn();
      // }
      box.onmousemove= function(){
          fn();
      }
  </script>
  ```

## 1.2 防抖debounce

- 什么是防抖

  当触发事件后,一定时间内没有再触发事件,才执行函数。主要用于输入频繁触发的场景,例如窗口尺寸调整

- debounce的特点是当事件快速连续不断触发时，一定时间内动作只会执行一次。

### 案例代码1: 窗口尺寸调整时

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <style>
      * {
        margin: 0;
        padding: 0;
      }

      #common {
        position: absolute;
        left: 0;
        top: 0;
        border: 1px solid #000;
        width: 300px;
        height: 600px;
      }

      #special {
        position: absolute;
        left: 400px;
        top: 0;
        border: 1px solid #000;
        width: 300px;
        height: 600px;
      }
    </style>
  </head>

  <body>
    <div id="common"></div>
    <div id="special"></div>
    <script>
      var delay = 300
      var common = document.getElementById('common') // 获取页面的左边
      var special = document.getElementById('special') // 获取页面的右边
      var timeid

      function debounce(fn, delay) {
        // 定义一个debounce函数
        clearTimeout(timeid)
        timeid = setTimeout(function () {
          fn()
        }, delay)
      }

      function addlist() {
        // 监听事件的响应事件，执行dom操作。
        special.innerHTML += '<li>k</li>'
      }

      function commonWay() {
        // 这是执行了普通的函数
        common.innerHTML += '<li>k</li>'
      }
      // 这是执行debounce的。
      window.onresize = function () {
        debounce(addlist, delay) //这一个仿抖设置
        commonWay() // 这个是普通的函数执行
      }
    </script>
  </body>
</html>
```

## 1.3 节流

- 预先设定一个执行周期，当调用动作的时刻大于等于执行周期才会执行一次该动作，然后进入下一个新周期。

- 节流： 降低触发的频率。比如：onmousemove原本1s触发100次，现在让它1s触发20次

  案例代码

  ```html
  <!DOCTYPE html>
  <html lang="en">
    <head>
      <meta charset="UTF-8" />
      <meta http-equiv="X-UA-Compatible" content="IE=edge" />
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <title>Document</title>
      <style>
        * {
          margin: 0;
          padding: 0;
        }

        .box {
          width: 300px;
          height: 1300px;
          border: 1px solid #000;
        }
      </style>
    </head>

    <body>
      <div class="box"></div>

      <script>
        function func() {
          console.log('滚动的业务代码')
        }
        function throttle(func, wait) {
          let timestamp
          return function () {
            let now = Date.now()
            if (!timestamp || now - timestamp > wait) {
              func.apply(this, arguments)
              timestamp = now
            }
          }
        }

        const throttled = throttle(func, 300)
        window.onscroll = throttled
      </script>
    </body>
  </html>
  ```

## 1.4 面试回答

- 什么是防抖和节流

1. window.onscroll onresize onmousemove(拖拽效果) onkeyup(搜索框的按键)这些事件频繁被触发，导致重复执行DOM操作、资源加载等重行为、会导致UI停顿甚至浏览器崩溃。
2. 防抖(debounce)和节流(throttle)都是用于控制函数调用频率的技术。它们都有助于优化页面性能,改善用户体验
3. 防抖是当触发事件后,一定时间内没有再触发事件,才执行函数
4. 节流是每隔一定时间只执行一次函数，从而降低触发的频率。比如：onmousemove原本1s触发100次，现在让它1s触发20次

- 写出防抖函数

```js
function debounce(func, wait) {
  let timeout
  return function () {
    clearTimeout(timeout)
    timeout = setTimeout(() => {
      // arguments第一个参数为事件对象
      func.apply(this, arguments)
    }, wait)
  }
}
```

写出节流函数

```js
function throttle(func, wait) {
  let timestamp
  return function () {
    let now = Date.now()
    if (!timestamp || now - timestamp > wait) {
      func.apply(this, arguments)
      timestamp = now
    }
  }
}
```

# 2、回流和重绘 （高频面试题）

## 什么是回流和重绘

回流(reflow)和重绘(repaint)是浏览器进行页面渲染的两个重要概念

- 回流

  回流指的是元素的尺寸、布局、位置发生变化,导致页面的布局需要重新计算。

  回流会触发浏览器重新计算元素的几何信息,并且重新排布页面元素的位置。

- 重绘

  重绘指的是元素的外观发生变化,但尺寸和布局不变,只需要重新绘制元素。

  重绘不会影响页面的布局,只会更新元素的外观。

  > 页面初始化一定会有一次回流和重绘，以后重绘不会引起回流，但回流一定引起重绘

## 触发回流和重绘的场景

- 添加/删除可见元素:会引发回流和重绘。
- 改变元素位置/尺寸:会引发回流和重绘。
- 改变元素外观(颜色等):只会引发重绘。 -
- JavaScript 动态设置元素样式:可能引发回流和重绘。

频繁的回流和重绘会对性能产生很大影响。我们应该尽量减少由JavaScript代码触发的回流和重绘。

## 如何避免回流和重绘

1. 避免频繁读取 offsetTop、offsetLeft、offsetWidth 等引起回流的属性。可以考虑使用元素的初始值,然后在回流后再次更新实际值。
2. 尽量通过 CSS 预先定义元素样式,而不是在 JavaScript 中频繁设置样式。
3. 可以考虑使用文档碎片来添加多个元素,然后一次性添加到文档中。
4. 使用 requestAnimationFrame 来定时执行动画,它可以优化多个回流重绘。
5. 避免频繁读取布局属性 scrollTop、scrollLeft 等。 可以考虑缓存值,限制读取频率。
6. CSS 动画开启硬件加速可以满足一定的动画需求而不引起回流。

> 所以,理解回流和重绘的概念,知道它们可能被触发的情况,并采取相应优化措施,可以有效地提高页面的渲染性能。这也是前端性能优化的一个重要方面。

## 案例： 理解 文档片段

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document</title>
    <style>
      * {
        margin: 0;
        padding: 0;
      }

      .box {
        width: 300px;
        height: 300px;
        border: 1px solid #000;
      }
    </style>
  </head>

  <body>
    <div class="box">
      <ul>
        <li>111111</li>
        <li>222222</li>
      </ul>
    </div>

    <button>单击按钮，动态添加5个li</button>
    <script>
      let btn = document.querySelector('button')
      let boxUl = document.querySelector('.box ul')
      btn.onclick = function () {
        // 反复操作dom
        console.time('时间1')
        for (let i = 3; i < 8; i++) {
          let liEl = document.createElement('li')
          let txt = document.createTextNode(i)
          liEl.appendChild(txt)
          boxUl.appendChild(liEl)
        }
        console.timeEnd('时间1')

        //操作一次dom
        console.time('时间2')

        //文档片段存在于内存中，并不在DOM树中，
        // 所以将子元素插入到文档片段时不会引起页面回流（对元素位置和几何上的计算）。
        // 因此，使用文档片段通常会带来更好的性能。

        //创建文档片段
        let fragment = document.createDocumentFragment()
        for (let i = 8; i < 13; i++) {
          let liEl = document.createElement('li')
          let txt = document.createTextNode(i)
          liEl.appendChild(txt)
          fragment.appendChild(liEl)
        }
        boxUl.appendChild(fragment)
        console.timeEnd('时间2')
      }
    </script>
  </body>
</html>
```

# 3. 编码和解码函数

## 为什么要使用编码函数

场景一： cookie中有特殊字符时

```js
// 当字符串中有特殊字符，没有编码时

let username = 'uu;aa=123'
// document.cookie="account=uu;aa=123"
// 结果只写入了account=uu
document.cookie = `account=${username}`
console.log(document.cookie) // "account=uu"
```

```js
// 当字符串中有特殊字符，编码时

let username = 'uu;aa=123'
document.cookie = `account=${encodeURIComponent(username)}`
console.log(document.cookie) // account=uu%3Baa%3D123
console.log(decodeURIComponent(document.cookie)) //account=uu;aa=123
```

场景二： url地 址中有特殊字符串时

```js
//向后端接口传参时，参数中有特殊字符

let username = 'uu&aa=123'
// url = "http://localhost:3000?username=uu&aa=123"
// 原本传一个键值对,结果后端会解析为两个键值对
fetch(`http://localhost:3000?username=${username}`)
  .then((res) => res.json())
  .then((res) => {
    console.log(res)
  })
```

## 常用的编码函数

- 编码函数 encodeURI() 解码函数decodeURI()

  encodeURI不编码字符有82个：!，#，$，&，'，(，)，\*，+，,，-，.，/，:，;，=，?，@，\_，~，0-9，a-z，A-Z

- 编码函数 encodeURIComponent() 解码函数decodeURIComponent()

  encodeURIComponent不编码字符有71个：!， '，(，)，\*，-，.，\_，~，0-9，a-z，A-Z

- escape和unescape已废弃不用

> encodeURIComponent用得最多， 可以根据自已的实际需求符合哪种编码来选择编码函数
