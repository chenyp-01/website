## 1. js对象

- JavaScript 中的所有事物都是对象：字符串、数值、数组、函数...此外，JavaScript 允许自定义对象。
- JavaScript 提供多个内建对象，比如 String、Date、Array 等等。
- 对象只是带有属性和方法的特殊数据类型。属性是与对象相关的值。方法是能够在对象上执行的动作。、
- 通过 JavaScript，您能够定义并创建自己的对象

## 2. 自定义对象的三种方式

### 2.1 用new Object实例化一个对象 （了解即可）

缺点： 不能用于重复创建具有相同模型（属性和方法）的多个对象

```js
//需求： 定义两个人的信息

let one = new Object() //空对象
one.name = 'aaa'
one.age = 12
one.address = '开封'
one.say = function () {
  console.log('我来自开封')
}
one.say()

let two = new Object() //空对象
two.name = 'nnn'
two.age = 13
two.address = '郑州'
two.say = function () {
  console.log('我来自郑州')
}
two.say()
```

### 2.2 对象字面量 (json对象)

缺点：不能用于重复创建具有相同模型（属性和方法）的多个对象。

```js
// 以下三种写法完全等价
// demo1:
let one = { name: 'a', age: 12 }

// demo2
let one = {} //和new Object()等价，创建空对象
one.name = 'a'
one.age = 12

// demo3
let one = {
  name: 'a',
  age: 12,
}
```

### 2.3 使用构造函数( 对象构造器）自定义对象

```js
function Student(name, age, address) {
  this.name = name
  this.age = age
  this.address = address
  this.say = function () {
    console.log('我是', this.name, '我来自', this.address)
  }
}

// new出来的新对象称为实例化对象
let one = new Student('Alice', 11, '郑州')
one.say()

let two = new Student('Jenny', 13, '开封')
two.say()
```

## 3. 构造函数

### 3.1 什么是构造函数

当通过 `new` 关键字调用一个函数时，这个函数就可以称为构造函数

new关键字的作用是生成一个实例化对象

### 3.2 new一个实例化对象经历的四个阶段 （面试题）

1. 创建一个空的实例化对象
2. 让构造函数中的this指向空的实例化对象，
3. 执行（调用 ）构造函数，从而创建实例化对象自身的属性和方法
4. 返回实例化对象

### 3.3 构造函数与普通函数的区别

- 调用方式不同

​ 普通函数只用 `**函数名**` 调用

​ 构造函数通过 `**new 关键字**` 调用

- 返回值不同

​ 普通函数的返回值是函数体内 return 的结果

​ 构造函数内部不需要return语句，会自动返回 new 关键字 生成的实例化对象

- 其它不同

  构造函数首字母大写 （不大写也可以，只是不规范）

  构造函数内部会给this对象赋值，从而创建实例自身的属性和方法。 （没有this，实例化对象不能创建自身 的属性和方法）

```javascript
function fun() {
  return 1
}
var a = fun()
var b = new fun()
console.log(a) // 1
console.log(b)
```

## 4. 原型对象

### 4.1 原型对象 prototype

`**所有函数**`都有一个原型对象 prototype，并且`**只有函数**`才拥有原型对象 prototype。

原型对象也是一个对象,它内置两个属性\_proto\_\_ (注意两边各两个下划线)和 constructor

```js
function a() {}
console.log(a.prototype)
console.log(a.prototype.constructor)
console.log(a.prototype.__proto__)
```

### 4.2 构造函数 constructor (了解)

函数的原型对象的构造函数是这个函数自己。也就是

```javascript
function a() {
  console.log(1)
}
console.log(a.prototype.constructor === a) //true
```

### 4.3 实例化对象 （了解）

- 假如通过new 关键字从 `**函数fun**` 里创建的 `**对象obj**` ，可以把 obj 称为 fun 的实例化对象；
- 通过字面量方式创建的对象可以称为 `**Object**` 的实例化对象；
- 实例化对象和普通对象没有任何区别

### 4.4 **`\*\***proto**\*\*` **属性 （重点）

每个实例化对象都拥有一个属性 `**__proto__**` ,这是浏览器实现的属性，通过它可以访问到 构造函数上的 原型对象 prototype;

```javascript
function a() {
  console.log(1)
}
let b = new a()
console.log(b.__proto__ === a.prototype) // true
```

## 5. 原型链 （重点理解）

1.  原型对象： 所有的构造函数在初始化时，都会自动生成一个特殊的实例化对象，构造函数的prototype属性指向该对象，该对象称为原型对象，或prototype对象

2.  proto**属性： 所有构造函数new出来的实例化对象，都有一个**proto**属性，该属性指向构造函数的原型对象,原型对象也有自己的**proto属性，指向上一级构造函数的原型对象。 **proto**会一直找到Object的原型对象，Object原型对象的**proto**指向null.

3.  原型链： 由一系列**proto**属性串连起来的原型对象，称为原型链

4.  原型链的意义： 实例化对象在访问属性和方法时，会先访问自身的属性和方法，如果自身不存在对应的属性和方法，则会自动通过**proto**属性在整个原型链上查找，只要找到对应的属性和方法，就能正确执行，如果原型链上没找到，则报错

## 6. 利用构造函数实现封装

封装就是尽量把使用的方式简单化，使用的时候只需要知道参数和返回值，其他条件尽量不要人为进行设置

常见的对象封装方式

### 6.1 构造函数模式

```javascript
function Cat(class){
	this.class = class;
      this.sayHi = function(){
        console.log('Hi! I am'+this.class)
      }
}
var cat1 = new Cat('三花');
var cat2 = new Cat('蓝猫');
cat1.sayHi();// Hi! I am 三花;
cat2.sayHi(); Hi! I am 蓝猫;
```

这种方式有个不好的地方，每只cat都会单独创建一个sayHi 方法,这个方法应该是公共方法，不需要给每只cat都创建，可以使用 `**prototype**` 来优化它

### 6.2 构造函数+原型 模式

在构造函数的基础上，将私有属性写在构造函数中，将通用的方法定义在原型对象上，这样每个实例都会有自己的一份实例属性的副本，但同时又共享对方法的引用，最大限度的节省了内存。

```javascript
function Cat(class){
	this.class = class;
}
Cat.prototype.sayHi = function(){
	console.log('Hi! I am'+this.class)
}
```

## 7. 与对象相关的操作

- in运算符

  in运算符可以用来判断，某个实例是否含有某个属性或方法，不管是不是本地属性或方法

  ```js
  function Cat() {
    this.name = '小白'
  }

  Cat.prototype.move = function () {}

  var c = new Cat()

  console.log('move' in c) //true, 原型继承的也返回true   c.move
  console.log('name' in c) //true
  console.log('run' in c) //false

  // 移动端触屏事件
  document.ontouchstart = function () {}

  console.log(1)
  console.log('是否是移动端', 'ontouchstart' in document)
  ```

- hasOwnProperty()方法：

  判断一个属性是自身属性，还是从原型链继承的属性

  如果是自身存在的，返回真，否则返回假

  ```js
  function Cat() {
    this.name = '小白'
  }

  Cat.prototype.move = function () {}
  var c = new Cat()

  console.log(c.hasOwnProperty('name')) //true
  console.log(c.hasOwnProperty('move')) //false

  // demo
  Object.prototype.color = 'red'

  let obj = {
    name: 'Alice',
    age: 12,
    address: '郑州',
  }

  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      console.log(key, obj[key])
    }
  }
  ```

- instanceOf : 判断一个对象是否为另一个对象的实例

  ```js
  let btnList = document.querySelectorAll('button')

  //   第一种判断方法 instanceOf
  let isArray1 = btnList instanceof Array

  //   第二种判断方法 ES6的 Array.isArray()
  let isArray2 = Array.isArray(btnList)
  console.log(isArray2)
  ```
