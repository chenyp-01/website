## 1. 了解闭包的应用场景

需求： 实现一个计数器 （抽奖计数, 投票计数）

```html
<p>
  <button>
    抽奖次数:
    <span>0</span>
  </button>
</p>
<p>
  <button>
    投票次数:
    <span>0</span>
  </button>
</p>
```

### 第一个版本： v1.0

用两个变量统计次数

缺点： 代码冗余

```js
let btn = document.querySelectorAll('button')
let span = document.querySelectorAll('span')
//抽奖次数
let num1 = 0

btn[0].onclick = function () {
  num1++
  span[0].innerHTML = num1
}

//投票次数
let num2 = 0
btn[1].onclick = function () {
  num2++
  span[1].innerHTML = num2
}
```

### 第二个版本： v2.0

更新迭代, 封装代码，使用一个全局变量计数

\*\* 使用全局变量，无法实现需求

\*\* 原因：全局变量的缺点： 容易被污染

```js
// 补充： dom.children可以返回dom的所有子标签元素的dom集合

let btn = document.querySelectorAll('button')
let span = document.querySelectorAll('span')

var num = 0 //全局变量 ****************全局变量的缺点： 容易被污染

function counter() {
  num++
  this.children[0].innerHTML = num
}

//无意中更改了num
num = 8

//抽奖
btn[0].onclick = counter

//投票
btn[1].onclick = counter
```

### 第三个版本： v3.0

更新迭代， 封装代码， 使用局部变量计数

\*\* 局部变量的缺点： 在函数运行完毕即销毁，不能实现累加

```js
let btn = document.querySelectorAll('button')
let span = document.querySelectorAll('span')

function counter() {
  var num = 0 // 局部变量
  num++
  this.children[0].innerHTML = num
}

//抽奖
btn[0].onclick = counter

//投票
btn[1].onclick = counter
```

### 第四个版本：V4.0

用闭包解决

```js
let num

function counter() {
  let num = 0 // 局部变量
  function a() {
    console.log(1)
  }

  // 下面的函数表达式即为闭包
  return function () {
    num++
    this.children[0].innerHTML = num
  }
}
num = 8
//抽奖
btn[0].onclick = counter()
// a() 父函数内部的子函数在父函数外部不能访问
//投票
btn[1].onclick = counter()
```

## 2. 什么是闭包

- 闭包(Closure)是JavaScript中一个很强大的特性。闭包就是能够读取其他函数（父函数）内部变量的函数。
- 简单来说,闭包就是一个函数内部的 一个子函数,这个内部函数有权访问外部函数的变量和参数,即使外部函数已经关闭。

## 3. 写一个简单的闭包 （面试题）

```js
function a() {
  var b = 9
  return function () {
    console.log(b)
  }
}
var c = a()
c()
c = null //根据需要，销毁闭包函数，避免内存泄露
```

## 4. 闭包的优缺点 （面试题）

优点:

1.  可以避免全局变量的污染。将变量封闭在函数内部,只能通过闭包访问。
    2.  使变量的值长期保存在内存中。闭包可以访问外部函数定义的变量,外部函数调用结束后,其内部变量依然存在。
    3.  私有成员的模拟。可以将变量和函数封闭在闭包内,对外部隐藏内部的细节。

缺点:

1. 闭包过多会导致内存泄漏。保存在闭包中的变量长期不被释放会占用内存。
   2. 闭包中的变量在外部函数调用结束后不会被释放,对系统性能有影响。

## 6. 闭包面试题二

```js
let person = (function () {
  let name = 'default'
  return {
    getName: function () {
      return name
    },
    setName: function (newName) {
      name = newName
    },
  }
})()
console.log(person.name) // undefined
console.log(person.getName()) //default
person.setName('rice')
console.log(person.getName()) // rice
```
