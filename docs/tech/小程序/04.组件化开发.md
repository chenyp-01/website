# 1.创建新组件
1. 在根目录创建components文件夹
2. 创建对应页面的子文件夹 （例： indexpage---存放首页中相关的组件）
3. 继续在indexpage内创建文件夹 (例： tab---定义tab组件的文件夹)
4. 在tab上右击--> 新建Component-->输入名字-->回车后会自动生成四个文件

tab.js

tab.json

tab.wxml

tab.wxss

# 2.在页面中引入组件
以index为例， 在index页面中引入该组件<br />在index.json中配置
```json
"usingComponents": {
  "x-tab": "/components/indexpage/tab/tab"
},
```
"x-tab": 为自定义组件的名称"/components/indexpage/tab/tab"： 为组件文件的路径（注意第一个tab 为文件夹，第二个tab为文件名）<br />在index.wxml中使用组件
```html
<x-tab></x-tab>
```
# 3.编辑组件

1. tab.json: 组件中也可引入自己的子组件
2. tab.wxml 组件的模板
3. tab.wxss 组件的样式
4. 重点：tab.js 可实现父子组件传值

Component() : 创建自定义组件，接受一个 Object 类型的参数。<br />常用参数：<br />1、properties: { } 属性定义，即从父组件接收的数据<br />2、data: {} 私有数据，可用于模板渲染<br />3、 methods: {} 处理函数<br />4、 observer: { } 监听数据的改变
# 4.页面向组件传值

1. 页面通过自定义属性向组件传值： <x-tab 自定义属性=”值”>
2. 组件通过properties接收页面传来的值 （可在组件的模板中直接使用）
```javascript
properties: {
  msg: String,
    goodsId: {
    type: Number, // 数据类型
      value: 4 // 默认值
  }
},
```

```html
  </view>
  <!-- <x-tab goodsId="{{list[cur].id}}" type="msg"></x-tab> -->
</view>
```
index.js
```javascript
Page({
  data: {
    list: [],
    cur: 0
  },
  onLoad(){
    wx.request({
      url: 'http://localhost:3000/tabData',
      method: 'get',
      success:(res)=>{
        console.log(res.data);
        this.setData({
          list: res.data
        })
      }
    })
  },
  toggle(e){
    let index  = e.target.dataset.index
    this.setData({
      cur: index
    })
  }
})
```

# 7.组件中生命周期函数
组件的生命周期，指的是组件自身的一些函数，这些函数在特殊的时间点或遇到一些特殊的框架事件时被自动触发。<br />其中，最重要的生命周期是 created attached detached ，包含一个组件实例生命流程的最主要时间点。

- 组件实例刚刚被创建好时， created 生命周期被触发。此时，组件数据 this.data 就是在 Component 构造器中定义的数据 data 。 **此时还不能调用 setData 。** 通常情况下，这个生命周期只应该用于给组件 this 添加一些自定义属性字段。
- 在组件完全初始化完毕、进入页面节点树后， attached 生命周期被触发。此时， this.data 已被初始化为组件的当前值。这个生命周期很有用，绝大多数初始化工作可以在这个时机进行。
- 在组件离开页面节点树后， detached 生命周期被触发。退出一个页面时，如果组件还在页面节点树中，则 detached 会被触发。
```javascript
Component({
  lifetimes: {
    created: function() {
    // 在组件实例刚刚被创建时执行
    },
    attached: function() {
      // 在组件实例进入页面节点树时执行
    },
    detached: function() {
      // 在组件实例被从页面节点树移除时执行
    },
  },
  pageLifetimes: {
    // 组件所在页面的生命周期函数
    show: function () { },
    hide: function () { },
  }
})
```
# 8.组件中的数据监听器
数据监听器可以用于监听和响应任何属性和数据字段的变化。有时，在一些数据字段被 setData 设置时，需要执行一些操作。例如， this.data.sum 永远是 this.data.numberA 与 this.data.numberB 的和。此时，可以使用数据监听器进行如下实现。
```javascript
Component({
  attached: function() {
    this.setData({
      numberA: 1,
      numberB: 2,
    })
  },
  observers: {
    activeIndex: ()=>{
        console.log('activeIndex发生了变化');
    },
    'numberA, numberB': function(numberA, numberB) {
      // 在 numberA 或者 numberB 被设置时，执行这个函数
      this.setData({
        sum: numberA + numberB
      })
    }
  }
})
```
# 9.组件中的插槽
1.只有一个插槽
```html
// 组件：

<view>
  .....
  <slot></slot>
</view>

// 页面
<组件标签>
  <view>插入的内容</view>
</组件标签>
```
2.多个插槽组件
```html
<view>
  .....
  <slot name=“c1”></slot>
  <slot name=“c2”></slot>
</view>
```
页面
```html
<组件标签>
  <view slot=”c1”>插入的内容</view>
  <view slot=”c2”>插入的内容</view>
</组件标签>
```
```javascript
Component({
  options: {
    multipleSlots: true // 在组件定义时的选项中启用多slot支持
  },
  properties: { /* ... */ },
  methods: { /* ... */ }
})
```
