# 5.面试回答：
## 简答  
下单后，后台接口返回一些参数，我用wx.requestPayment向微信发起支付请求，携带后台的参数，用户支付成功后，显示成功页面即可
## 完整回答
1.打开小程序，点击直接下单。<br />2.wx.login获取用户临时登录凭证code，发送到后端服务器换取openId。<br />3.在下单时，小程序需要将购买的商品Id，商品数量，以及用户的openId传送到开发者服务器。<br />4.开发者服务器在接收到商品Id、商品数量、openId后，生成服务器订单数据（存入订单表），同时经过一定的签名算法，向微信支付发送请求，获取预付单信息(prepay_id)，同时将获取的数据再次进行相应规则的签名，向小程序端响应必要的信息。<br />5.小程序端在获取对应的参数后，调用wx.requestPayment()发起微信支付，唤醒支付工作台，进行支付。<br />6.接下来的一些列操作都是由用户来操作的包括了微信支付密码，指纹等验证，确认支付之后执行鉴权调起支付。<br />7.鉴权调起支付：在微信后台进行鉴权，微信后台直接返回给前端支付的结果，前端收到返回数据后对支付结果进行展示。<br />8.推送支付结果：微信后台在给前端返回支付的结果后，也会向后台（商家后台）也返回一个支付结果，后台通过这个支付结果来更新订单的状态。
# 6.前端实现支付的代码
参考： [https://blog.csdn.net/qq_47272950/article/details/130865505](https://blog.csdn.net/qq_47272950/article/details/130865505)<br />参考案例用uniapp实现，只需把API前缀的uni改为wx,即为下方代码
```javascript
//支付函数
goPay() {
    console.log('单击立即充值或提交订单后,调用该方法');

    //向开发者服务器发送下单请求
    wx.request({
        url: app.globalData.URL + "/createStoreGoodsOrder",  // 下单接口，会返回预付单信息
        method: 'POST',
        data: {
            goodsId: 23, //商品id
            count: 2, //商品数量
            // 其它订单所需信息
            // ......
            opendid: '', // 用户登录后拿到的openid
            token: '', // 用户登录后拿到的token 
        },
        success(res) {
            //如果下单失败，则弹出提示并返回false
            if (res.data.status == 'no') {
                //弹出下单失败的提示
                wx.showToast({ title: res.data.msg, icon: "none" });
                return false;
            }

            // 如果不需要支付，此时开发者服务器已经创建订单信息, 则跳转到订单详情页并返回false

            if (res.data.data.is_need_pay == 0) {
                wx.redirectTo({
                    url: './localLifeOrderDetail?store_order_id=' + res.data.data.goods_order_id
                })
                return false;
            } else {

                //如果需要支付，则res中已经携带了调用微信支付接口进行支付的相关参数
                wx.requestPayment({
                    timeStamp: res.data.data.pay_data.timeStamp,// 时间戳
                    nonceStr: res.data.data.pay_data.nonceStr, // 随机字符串
                    package: res.data.data.pay_data.package, // 统一下单接口返回的 prepay_id 参数值
                    signType: 'MD5', // 签名算法，应与后台下单时的值一致
                    paySign: res.data.data.pay_data.paySign, // 签名
                    appId: app.globalData.appid,
                    success: function (ress) {
                        //支付成功后，弹出提示并在2秒后跳转到订单详情页
                        wx.showToast({
                            title: '成功',
                            duration: 2000
                        });
                        setTimeout(function () {
                            wx.redirectTo({
                                url: './localLifeOrderDetail?store_order_id=' + res.data.data.store_order_id
                            })
                        }, 2000)
                    },
                    fail: function (err) {
                        //支付失败后，直接跳转到订单详情页
                        wx.redirectTo({
                            url: './localLifeOrderDetail?store_order_id=' + res.data.data.store_order_id
                        })
                    }
                });
            }
        }
    })
}
```
